# 04 订单承接 SOP

## P0 成功定义

P0 成功必须同时满足：

1. 客户是陌生真实客户。
2. 客户主动从渠道进入咨询或下单。
3. 客户完成付款。
4. 商品完成交付。
5. 有完整证据：聊天、商品页/广告、付款、发货、签收/交付、售后。

## 不算 P0 成功

| 情况 | 处理 |
|---|---|
| 朋友/合作方/本地联系人自己买 | 渠道信号，不算 P0 |
| 小卖家/分销商买小批 | P3 线索，不算 C 端 P0 |
| COD 全额货到付款 | 只算弱信号，不算 P0 |
| 口头预订/加购/私聊说想买 | 不算订单 |
| 只付少量订金，尾款 COD | 算 P0-lite，不算正式 P0 |
| 平台/广告有询盘但没付款 | 只算渠道数据 |

## 可接受付款形式

| 付款形式 | 是否算 P0 | 备注 |
|---|---|---|
| 平台内在线付款 | 是 | 最强 |
| 本地 payment link | 是 | 需保留支付截图和交易号 |
| 本地银行转账/钱包转账 | 是 | 需核对收款人、时间、金额 |
| 全额预付到合作方账户 | 是 | 需合作方对账 |
| 订金 + 尾款 COD | P0-lite | 可记录，但不作为主成功 |
| 全额 COD | 否 | 拒收和伪需求风险高 |

## 标准订单流程

```text
1. 客户从渠道进入
2. 客服确认 SKU、价格、配送城市、付款方式
3. 客户确认订单
4. 发送付款方式和退换规则
5. 客户付款
6. 记录付款证据
7. 生成订单编号
8. 仓库/合作方发货
9. 记录物流单号或交付证明
10. 客户签收/确认
11. 售后跟进一次
12. 复盘订单
```

## 订单编号

格式：

```text
CA-{country}-{yyyymmdd}-{channel}-{sku}-{sequence}
```

示例：

```text
CA-KZ-20260615-OLX-CARHOLDER-001
```

## 私聊话术结构

不要一上来长篇介绍。按顺序问：

1. 你在哪个城市？
2. 你要哪个颜色/款式？
3. 是否确认价格包含/不包含配送？
4. 付款后什么时候发货？
5. 退换规则是什么？

必写信息：

- 商品价格。
- 配送费。
- 发货时间。
- 是否有现货。
- 付款方式。
- 退款/补发规则。

## 证据留存

每个 P0/P1 订单必须保存：

| 证据 | 必须 |
|---|---|
| 渠道截图/链接 | 是 |
| 客户聊天截图 | 是 |
| 客户付款截图/交易号 | 是 |
| 内部订单记录 | 是 |
| 发货单/物流单号/交付照片 | 是 |
| 客户签收/确认 | 是 |
| 售后记录 | 是 |
| 退款/补发记录 | 如发生 |

使用模板：[templates/order_record.csv](templates/order_record.csv)。

## 风控规则

- 不接受高价值首单。
- 不承诺全境包邮。
- 不承诺包税，除非本地清关方明确报价并承担。
- 不用个人账户长期收款；P0 可以用，P1 必须切到主体/商户账户。
- 不收客户敏感证件，除非物流/清关明确要求且有合规存储方式。
