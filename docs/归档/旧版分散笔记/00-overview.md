# 00 Overview

## 目标

做一个可执行的中亚五国小宗商品出海电商 MVP SOP。

当前资源假设：

- 中国侧拿货不是问题。
- 资金和资源不是主要限制。
- 本地资源“有一些但未盘清”。
- 目标不是马上利润最大化，而是先完成 P0/P1。

## 阶段定义

| 阶段 | 核心问题 | 不该做的事 |
|---|---|---|
| P0 | 陌生客户愿不愿意付款，链路能不能交付 | 不追求五国铺开；不追求复杂品牌；不碰高合规 SKU |
| P1 | 这个闭环能不能重复 | 不只靠某个朋友/合作方人工硬推 |
| P2 | 能不能连续出单且利润稳定 | 不盲目加 SKU；不忽略退款/客服/退货成本 |
| P3 | 能不能接分销/批发 | 不在 C 端链路没跑通前承诺大批量 |

## 默认 MVP 路线

```text
国家：Kazakhstan 主线，Uzbekistan 备线
商品：非通电车内用品 + 通用手机周边
渠道：Classifieds / Instagram / Telegram / WhatsApp
承接：聊天确认 + 本地支付链接/转账
履约：小批量到本地节点或轻仓，再本地派送/自提
复盘：按订单证据、客户问题、履约成本、退款/补发记录判断是否进 P1
```

## 为什么不是五国并行

五国不是一个同质市场：

- Kazakhstan 电商和非现金支付最成熟。
- Uzbekistan 人口最大，增长快，但平台和本地化要求明显。
- Kyrgyzstan 可低成本测试，但市场小、现金化仍明显。
- Tajikistan 支付在改善，但履约和线上触达弱。
- Turkmenistan 支付、互联网、社媒和清关透明度不足，不适合 P0。

## 操作优先级修正规则

默认市场排序是：

```text
Kazakhstan > Uzbekistan > Kyrgyzstan > Tajikistan > Turkmenistan
```

但执行时按本地资源修正：

| 情况 | 修正 |
|---|---|
| Kazakhstan 没有可靠本地收款/派送节点，Uzbekistan 有 | Uzbekistan 可先跑 P0 |
| Kazakhstan 有 Kaspi Pay/本地银行卡/轻仓 | Kazakhstan 作为 P0 首站 |
| Kyrgyzstan 有强本地合作方，哈乌没有 | Kyrgyzstan 可做低成本 P0，但不默认作为 P1 主站 |
| Tajikistan/Turkmenistan 只有普通联系人，没有收款/清关/派送能力 | 不进入 P0 |

## 交付等级

当前 SOP 定位：🟡 可用。

它能指导团队执行 P0/P1，但还不是生产级跨境贸易合规系统。升级到 🔴 结实 需要：

- 每个 SKU 的 HS code、税率、认证路径由专业清关/合规方确认。
- 每个国家至少 2 个支付/履约备选。
- 有真实订单、退款、物流时效、客服数据回填。
- 平台入驻和本地主体结构完成法律/税务确认。
