# 中亚五国小宗商品出海电商调研报告与名词梗概

数据冻结日期：2026-06-15  
写作目的：让没有跨境电商经验的人，先看懂市场、关键概念、风险和为什么这么选。

## 一句话结论

中亚五国不是一个统一市场。MVP 不应该五国同时做，应该先选一个最容易完成“陌生客户主动付款 + 商品交付”的国家。当前默认选择是 **Kazakhstan**，备选是 **Uzbekistan**。

## 我们要验证什么

这次 MVP 的核心不是利润最大化，也不是先做品牌。

| 阶段 | 要验证的问题 | 通过标准 |
|---|---|---|
| P0 | 陌生客户是否愿意主动付款，订单链路能否闭环 | 至少 1 个陌生客户预付款/线上付款，完成交付，证据完整 |
| P1 | 同一个闭环能不能复制 | 同国家同链路能重复 10 单，或 2 个 SKU 各完成 3 单 |
| P2 | 能不能稳定出单并赚钱 | 连续 30 天有订单，扣除广告、补发、退款、客服、仓配后毛利为正 |
| P3 | 能不能做分销/批发 | 有分销客户、报价、样品、账期/预付、交付规则 |

## 五国市场判断

| 排名 | 国家 | 适合程度 | 结论 |
|---:|---|---|---|
| 1 | Kazakhstan | 高 | 首站默认。电商、支付、平台、消费力、数字化综合最强 |
| 2 | Uzbekistan | 高 | 第二市场。人口最大，增长快，支付生态清楚，但社媒渗透低于 Kazakhstan |
| 3 | Kyrgyzstan | 中 | 可低成本跟进。数字化不错，但市场小、电商规模公开证据弱 |
| 4 | Tajikistan | 低中 | 只做小样验证。支付改善，但线上触达和履约不稳 |
| 5 | Turkmenistan | 低 | P0 排除。支付、社媒、互联网、清关透明度都弱 |

### Kazakhstan 为什么排第一

| 事实 | 含义 |
|---|---|
| 2025 线上零售约 3.7688 万亿 KZT，占零售 14.3% | 用户线上购物习惯已经比较成熟 |
| 平台型交易占线上零售 86% | Marketplace 心智强 |
| 互联网渗透约 93.4%，社媒约 80.8% | 可用线上渠道触达用户 |
| 银行卡交易非现金按笔数约 98.48% | 用户更容易接受非现金付款 |
| Kaspi、Wildberries、Ozon、AliExpress 等平台成熟 | P1/P2 有承接渠道 |

### Uzbekistan 为什么排第二

| 事实 | 含义 |
|---|---|
| 人口约 37.2M，五国最大 | 潜在市场大 |
| 互联网用户约 33.1M | 可线上触达的人多 |
| 2024 电商约 $1.2B，占零售 3.8% | 增长空间大，但成熟度不如 Kazakhstan |
| Uzum、Payme、Click、Freedom Pay 等生态可查 | 本地平台和支付路径相对清楚 |
| Courier 个人限额 $200/月/收件人，postal $100 | 小包试单规则更明确 |

### Kyrgyzstan/Tajikistan/Turkmenistan 的位置

| 国家 | 可以做什么 | 不建议做什么 |
|---|---|---|
| Kyrgyzstan | 用 Lalafo、Telegram、Instagram 做低成本小样测试 | 不作为第一主战场，不一开始重投广告 |
| Tajikistan | 用 Somon、Alif、本地合作方测 5-10 单 | 不作为第一主战场，不承诺稳定全境履约 |
| Turkmenistan | 后续有本地进口商/分销商再看 | 不做 P0 D2C，不做中国直发真实客户首单 |

## 品类判断

用户已有方向是消费电子和车内用品。结论：**P0 不要从复杂消费电子开始，要从非通电车内/手机周边小件开始。**

| 品类 | P0 判断 | 原因 |
|---|---|---|
| 普通车载支架 | 优先 | 小、轻、直观、低合规，适合视频展示 |
| 车内收纳/挂钩/理线夹 | 优先 | 售后简单，不强依赖车型 |
| 防滑垫/临时停车牌/小内饰 | 优先/观察 | 低风险，但同质化强 |
| 通用手机桌面/车载支架 | 优先 | 用户理解成本低 |
| 手机壳/钢化膜 | 观察 | 机型管理复杂，容易发错货 |
| 数据线 | 观察 | 快充虚标、断线、接口兼容问题 |
| 车充/充电头 | P1 后再看 | 电气安全、发热、认证和售后风险 |
| 蓝牙耳机/音箱/无线 CarPlay | P0 避开 | 电池、无线、固件、兼容、退货都复杂 |
| 小家电 | P0 避开 | 插头、电压、认证、破损、保修复杂 |
| 功能性汽配 | P0 避开 | 车型匹配和安全责任复杂 |

## 渠道判断

| 渠道 | 适合阶段 | 说明 |
|---|---|---|
| OLX.kz / OLX Uzbekistan / Lalafo / Somon | P0 | 分类信息平台，适合快速获得陌生客户询盘 |
| Instagram / TikTok / Telegram / WhatsApp | P0 | 适合展示短视频、私聊承接、人工成交 |
| Kaspi / Uzum / Olcha / Flip | P1 | 信任强，但需要本地主体、资料、平台审核 |
| Wildberries / Ozon | P1/P2 | 可跨国复制，但结算、仓配、退货和主体要求复杂 |
| 独立站 | P1/P2 | 适合品牌资产、FAQ、复购、批发询盘，不适合作为 P0 唯一路径 |
| 分销/批发 | P3 | C 端链路验证后再做更稳 |

## 为什么 P0 不先做 Kaspi/Uzum

Kaspi 和 Uzum 是好渠道，但不一定适合“第一单之前”。

| 平台 | 问题 |
|---|---|
| Kaspi | 需要 Kaspi Pay、本地经营主体/账户、近期收款记录等条件 |
| Uzum | 需要乌兹注册主体/个体/自雇、注册材料、税务配置 |
| Wildberries/Ozon | 本地主体、税号、银行账户、仓配、退货、规则都更重 |

所以 P0 的合理路径是：

```text
先用 classifieds / 社媒找到陌生客户
-> 私聊确认商品和配送
-> 本地 payment link / 转账付款
-> 本地节点发货
-> 记录证据
```

## 名词梗概

### MVP

MVP = 最小可行版本。  
在这里指：不用做完整公司系统，只做能验证“陌生客户会付款、货能送到”的最小链路。

具体例子：

```text
1 个国家 + 1 个城市 + 5 个 SKU + 2 个渠道 + 1 个收款方式 + 1 个发货节点
```

### P0 / P1 / P2 / P3

| 名词 | 意思 |
|---|---|
| P0 | 第一优先级：完成首批真实陌生客户订单 |
| P1 | 第二优先级：把 P0 闭环复制成 SOP |
| P2 | 第三优先级：稳定出单和稳定利润 |
| P3 | 第四优先级：接触分销/批发客户 |

### 小宗商品

小宗商品 = 单件价值、重量、体积都比较小的商品。  
这里优先指车内小件、支架、收纳件、手机周边。

### 大宗商品

大宗商品 = 大批量、货值高、贸易链条重的商品。  
这里暂时不做，因为它更依赖清关、账期、合同、仓储、资金周转和 B2B 信用。

### SKU

SKU = 一个具体可售商品单位。  
颜色、型号、尺寸不同，通常算不同 SKU。

具体例子：

```text
黑色出风口车载支架 = 1 个 SKU
银色出风口车载支架 = 另 1 个 SKU
```

### D2C

D2C = 直接卖给消费者。  
这里指我们直接把车载支架卖给当地车主。

### B2B

B2B = 卖给商家。  
这里指卖给当地车品店、小卖家、分销商。

### Marketplace

Marketplace = 有买家流量和平台交易规则的电商平台。  
这里包括 Kaspi、Uzum、Wildberries、Ozon、Olcha。

### Classifieds

Classifieds = 分类信息平台。  
用户发商品信息，买家私聊或电话联系，平台通常不负责完整交易托管。

具体例子：

```text
OLX.kz、Lalafo、Somon
```

### Payment link

Payment link = 支付链接。  
卖家生成一个付款链接，客户打开链接付款。

### PSP

PSP = Payment Service Provider，支付服务商。  
它负责在线收款、交易处理、对账等能力。

具体例子：

```text
Payme、Click、Freedom Pay、Halyk EPAY
```

### Acquiring

Acquiring = 商户收单。  
它让商家可以收银行卡、钱包、二维码、线上付款。

### COD

COD = Cash on Delivery，货到付款。  
P0 不把全额 COD 算成功，因为客户没有提前承担付款风险，拒收概率高。

### Prepayment

Prepayment = 预付款。  
P0 最好用全额预付款；最低也要订金 + 明确尾款规则。

### 3PL

3PL = 第三方物流服务商。  
它负责仓储、发货、派送、退件等部分能力。

### Light warehouse

Light warehouse = 轻仓。  
这里指在目标国放少量库存，不是完整大仓。

具体例子：

```text
先在 Almaty 放 50-100 件小商品，由合作方发货
```

### Last-mile delivery

Last-mile delivery = 末端配送。  
指商品进入目标城市后，送到客户手里的那一段。

### Reverse logistics

Reverse logistics = 逆向物流。  
指退货、换货、补发、维修这类售后物流。

### HS code

HS code = 商品海关编码。  
它影响税率、监管条件、申报名称和清关材料。

### Customs clearance

Customs clearance = 清关。  
指货物进目标国时，海关检查、申报、征税、放行的过程。

### De minimis / duty-free threshold

De minimis = 小额免税或低值简化门槛。  
在这里主要看个人小包金额和重量限制。

当前调研里：

| 国家 | 小包相关门槛 |
|---|---|
| Kazakhstan | 个人小包约 €200 / 31kg |
| Kyrgyzstan | EAEU 口径约 €200 |
| Uzbekistan | Courier $200/月/收件人；postal $100 |
| Tajikistan | 未找到可靠公开个人小包限额 |
| Turkmenistan | 未找到可靠公开电商小包限额 |

### DAP / DDU / DDP

| 名词 | 意思 | MVP 处理 |
|---|---|---|
| DAP | 卖家送到指定地点，进口税费通常由买家/收件方处理 | 可用，但要讲清楚税费 |
| DDU | 未完税交付，税费不包含在卖家报价里 | P0 可用，但不要承诺包税 |
| DDP | 完税交付，卖家承担进口税费和清关责任 | P0 不建议承诺 |

### EAEU

EAEU = Eurasian Economic Union，欧亚经济联盟。  
在本项目里，Kazakhstan 和 Kyrgyzstan 受 EAEU 规则影响。

### EAC

EAC = EAEU 相关合规标志。  
电子、电气、部分消费品进入 EAEU 市场时可能需要对应认证或声明。

### TR CU / TR EAEU

TR CU / TR EAEU = 技术法规。  
它规定某些商品进入 EAEU 市场前需要满足什么安全、EMC、材料或限制要求。

### EMC

EMC = 电磁兼容。  
带电路板、充电器、无线设备等可能涉及 EMC 要求。

### LVD / Low voltage

LVD = 低压电气安全要求。  
充电器、电源适配器、小家电这类商品可能触发。

### RoHS

RoHS = 有害物质限制。  
电子电气产品可能涉及材料限制。

### RF

RF = Radio Frequency，无线射频。  
蓝牙、Wi-Fi、GPS、SIM、FM、遥控设备都可能触发无线相关审批。

### UN38.3

UN38.3 = 锂电池运输测试要求。  
带锂电池产品运输前通常需要相关测试文件。

### MSDS

MSDS = Material Safety Data Sheet，材料安全数据表。  
电池、液体、化学品、清洁剂、胶水等可能需要。

### IATA DGR

IATA DGR = 国际航空运输协会危险品规则。  
锂电池、液体、化学品等空运时可能受它约束。

### GMV

GMV = 商品交易总额。  
它是成交额，不等于利润。

### TPV

TPV = Total Payment Volume，支付交易总额。  
常用于支付平台或金融科技公司。

### AOV

AOV = Average Order Value，平均客单价。  
计算方式：

```text
总销售额 / 订单数
```

### Gross margin

Gross margin = 毛利。  
计算方式：

```text
售价 - 商品成本 - 头程/尾程物流 - 包装 - 平台费 - 支付手续费 - 补发退款成本
```

P0 可以不赚钱，但必须记录真实毛利。

### CAC

CAC = Customer Acquisition Cost，获客成本。  
计算方式：

```text
广告费 / 付款客户数
```

### Lead

Lead = 潜在客户线索。  
比如一个客户私聊问价格，但还没付款。

### Qualified lead

Qualified lead = 有明确购买意向的线索。  
比如客户已确认城市、款式、价格和付款方式，只差付款。

## 核心风险

| 风险 | 为什么危险 | 怎么处理 |
|---|---|---|
| 把 COD 当成功 | 客户没提前付款，拒收风险高 | P0 只认预付款/线上付款/转账 |
| 一开始上复杂电子 | 合规、运输、售后、兼容问题叠加 | P0 只做机械小件 |
| 五国同时做 | 每国支付、语言、履约、平台都不同 | 先 1 国 1 城市 |
| 只看市场不看本地资源 | 没收款/发货就无法闭环 | 先填上下文采集表 |
| 只做调研不做订单证据 | 没法判断市场是否真的成立 | 每单留聊天、付款、物流、售后证据 |
