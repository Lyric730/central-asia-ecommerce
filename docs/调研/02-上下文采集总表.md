# 中亚电商 MVP 上下文采集总表

目的：只收集会改变执行方案的信息。不要先开很多分散表，先把这张表填完。

## 填表规则

- 能确定就填具体内容。
- 不确定就填“不确定”。
- 没有资源就填“无”。
- 不要为了好看编答案。
- 填完后只选 1 个主国家、1 个备选国家、5 个以内 P0 SKU。

## 填完后的决策规则

| 如果 | 就这样做 |
|---|---|
| Kazakhstan 有收款和轻仓 | 直接按 Kazakhstan 版执行报告跑 |
| Kazakhstan 没有收款，Uzbekistan 有 Payme/Click/轻仓 | 把执行报告里的国家改成 Uzbekistan，城市改成 Tashkent |
| 两国都没有收款 | 先找本地收款/合作方，不要先投广告 |
| 没有本地轻仓 | 先找 1 个合作方节点或 3PL，别直接承诺快速送达 |
| SKU 里带电池/无线/车充占多数 | 先砍 SKU，保留机械小件 |
| 不能做俄语/当地语客服 | 先做话术模板和机器翻译，不要上大量广告 |
