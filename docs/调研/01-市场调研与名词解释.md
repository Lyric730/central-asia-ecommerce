# 中亚五国小宗商品出海电商调研报告与名词梗概

数据冻结日期：2026-06-15  
写作目的：让没有跨境电商经验的人，先看懂市场、关键概念、风险和为什么这么选。

## 一句话结论

中亚五国不是一个统一市场。MVP 不应该五国同时做，应该先选一个最容易完成“陌生客户主动付款 + 商品交付”的国家。当前默认选择是 **Kazakhstan**，备选是 **Uzbekistan**。

## 我们要验证什么

这次 MVP 的核心不是利润最大化，也不是先做品牌。

| 阶段 | 要验证的问题 | 通过标准 |
|---|---|---|
| P0 | 陌生客户是否愿意主动付款，订单链路能否闭环 | 至少 1 个陌生客户预付款/线上付款，完成交付，证据完整 |
| P1 | 同一个闭环能不能复制 | 同国家同链路能重复 10 单，或 2 个 SKU 各完成 3 单 |
| P2 | 能不能稳定出单并赚钱 | 连续 30 天有订单，扣除广告、补发、退款、客服、仓配后毛利为正 |
| P3 | 能不能做分销/批发 | 有分销客户、报价、样品、账期/预付、交付规则 |

## 五国市场判断

| 排名 | 国家 | 适合程度 | 结论 |
|---:|---|---|---|
| 1 | Kazakhstan | 高 | 首站默认。电商、支付、平台、消费力、数字化综合最强 |
| 2 | Uzbekistan | 高 | 第二市场。人口最大，增长快，支付生态清楚，但社媒渗透低于 Kazakhstan |
| 3 | Kyrgyzstan | 中 | 可低成本跟进。数字化不错，但市场小、电商规模公开证据弱 |
| 4 | Tajikistan | 低中 | 只做小样验证。支付改善，但线上触达和履约不稳 |
| 5 | Turkmenistan | 低 | P0 排除。支付、社媒、互联网、清关透明度都弱 |

### Kazakhstan 为什么排第一

| 事实 | 含义 |
|---|---|
| 2025 线上零售约 3.7688 万亿 KZT，占零售 14.3% | 用户线上购物习惯已经比较成熟 |
| 平台型交易占线上零售 86% | Marketplace 心智强 |
| 互联网渗透约 93.4%，社媒约 80.8% | 可用线上渠道触达用户 |
| 银行卡交易非现金按笔数约 98.48% | 用户更容易接受非现金付款 |
| Kaspi、Wildberries、Ozon、AliExpress 等平台成熟 | P1/P2 有承接渠道 |

### Uzbekistan 为什么排第二

| 事实 | 含义 |
|---|---|
| 人口约 37.2M，五国最大 | 潜在市场大 |
| 互联网用户约 33.1M | 可线上触达的人多 |
| 2024 电商约 $1.2B，占零售 3.8% | 增长空间大，但成熟度不如 Kazakhstan |
| Uzum、Payme、Click、Freedom Pay 等生态可查 | 本地平台和支付路径相对清楚 |
| Courier 个人限额 $200/月/收件人，postal $100 | 小包试单规则更明确 |

### Kyrgyzstan/Tajikistan/Turkmenistan 的位置

| 国家 | 可以做什么 | 不建议做什么 |
|---|---|---|
| Kyrgyzstan | 用 Lalafo、Telegram、Instagram 做低成本小样测试 | 不作为第一主战场，不一开始重投广告 |
| Tajikistan | 用 Somon、Alif、本地合作方测 5-10 单 | 不作为第一主战场，不承诺稳定全境履约 |
| Turkmenistan | 后续有本地进口商/分销商再看 | 不做 P0 D2C，不做中国直发真实客户首单 |

## 品类判断

用户已有方向是消费电子和车内用品。结论：**P0 不要从复杂消费电子开始，要从非通电车内/手机周边小件开始。**

| 品类 | P0 判断 | 原因 |
|---|---|---|
| 普通车载支架 | 优先 | 小、轻、直观、低合规，适合视频展示 |
| 车内收纳/挂钩/理线夹 | 优先 | 售后简单，不强依赖车型 |
| 防滑垫/临时停车牌/小内饰 | 优先/观察 | 低风险，但同质化强 |
| 通用手机桌面/车载支架 | 优先 | 用户理解成本低 |
| 手机壳/钢化膜 | 观察 | 机型管理复杂，容易发错货 |
| 数据线 | 观察 | 快充虚标、断线、接口兼容问题 |
| 车充/充电头 | P1 后再看 | 电气安全、发热、认证和售后风险 |
| 蓝牙耳机/音箱/无线 CarPlay | P0 避开 | 电池、无线、固件、兼容、退货都复杂 |
| 小家电 | P0 避开 | 插头、电压、认证、破损、保修复杂 |
| 功能性汽配 | P0 避开 | 车型匹配和安全责任复杂 |

## 渠道判断

| 渠道 | 适合阶段 | 说明 |
|---|---|---|
| OLX.kz / OLX Uzbekistan / Lalafo / Somon | P0 | 分类信息平台，适合快速获得陌生客户询盘 |
| Instagram / TikTok / Telegram / WhatsApp | P0 | 适合展示短视频、私聊承接、人工成交 |
| Kaspi / Uzum / Olcha / Flip | P1 | 信任强，但需要本地主体、资料、平台审核 |
| Wildberries / Ozon | P1/P2 | 可跨国复制，但结算、仓配、退货和主体要求复杂 |
| 独立站 | P1/P2 | 适合品牌资产、FAQ、复购、批发询盘，不适合作为 P0 唯一路径 |
| 分销/批发 | P3 | C 端链路验证后再做更稳 |

## 渠道和平台具体展开

### 先理解：渠道不是一种东西

这里的“渠道”至少分 5 类：

| 渠道类型 | 是什么 | 本项目怎么用 |
|---|---|---|
| 分类信息平台 | 用户发商品信息，买家私聊/电话联系，平台通常不托管完整交易 | P0 用来快速拿陌生客户询盘 |
| 社媒内容渠道 | 用短视频、图片、帖子展示商品 | P0 用来测试卖点、场景、价格 |
| 聊天承接工具 | 客户从平台/社媒进来后，在聊天里完成问答、报价、付款引导 | P0 必须用，人工成交 |
| Marketplace | 平台有搜索、店铺、评价、支付、规则 | P1/P2 用，P0 不作为前置 |
| 独立站 | 自己的网站/商品页 | P1/P2 用来做品牌、FAQ、复购、批发询盘 |

P0 要先解决：

```text
哪里找陌生客户
-> 客户怎么问
-> 客户怎么信
-> 客户怎么付款
-> 货怎么送到
```

所以 P0 的渠道组合不是“选一个平台”，而是：

```text
分类信息/社媒获客 + Telegram/WhatsApp 承接 + 本地收款 + 本地派送
```

### P0 渠道：分类信息平台

分类信息平台适合小白团队，因为上手快，不需要先完成复杂店铺入驻。缺点是平台通常不帮你解决收款、物流、售后，所以你们要自己补齐交易链路。

| 平台 | 国家 | 适合卖什么 | 怎么用 | 优点 | 缺点 |
|---|---|---|---|---|---|
| OLX.kz | Kazakhstan | 车载支架、车内收纳、手机周边、本地现货小件 | 发商品帖，标题写俄语，放实拍图，留下 WhatsApp/Telegram 联系方式 | Kazakhstan P0 最优先；用户会主动搜索商品 | 不托管交易；付款和配送要自己安排 |
| OLX Uzbekistan | Uzbekistan | 同上，另可测手机周边、家电配件类需求 | 先用俄语/乌兹语标题做商品帖，私聊承接 | 能低成本测 Tashkent 需求 | 卖家信息、付款、退换规则要写清 |
| Lalafo.kg | Kyrgyzstan | 低价小件、车内用品、二手/新品混合测试 | 发商品，客户通过聊天/电话联系 | 适合低成本测试 Bishkek | 市场小，支付和配送要本地化 |
| Somon.tj | Tajikistan | 小样验证，不适合重投入 | 发少量商品，看是否有真实询盘 | 能测 Dushanbe/Khujand 需求 | 支付、清关、履约不稳 |
| TMCARS | Turkmenistan | 汽车相关线索，不适合普通小商品主线 | 只作为汽车/车品信息渠道观察 | 和汽车场景相关 | Turkmenistan 不适合 P0 D2C |

分类信息平台的核心动作：

1. 标题写用户会搜的词，不写品牌愿景。
2. 第一张图必须是真实商品场景图。
3. 描述里写清城市、价格、配送、付款方式。
4. 所有询盘导到 Telegram/WhatsApp 统一承接。
5. 记录客户问了什么、为什么不付款。

错误用法：

| 错误 | 后果 |
|---|---|
| 只发供应商白底图 | 信任弱，像搬运/诈骗 |
| 不写城市和配送 | 客户会反复问，客服耗时高 |
| 不写付款方式 | 询盘多但付款少 |
| 标题堆关键词 | 平台可能降权，用户也不信 |
| 发太多 SKU | 小团队客服和履约会乱 |

### P0 渠道：Instagram / TikTok / Facebook

社媒不是用来“立刻做品牌”的，P0 只用它验证三个问题：

```text
用户会不会停下来看片
用户会不会问价
用户会不会被引导到聊天并付款
```

| 渠道 | 适合国家 | 适合内容 | P0 用法 | 注意 |
|---|---|---|---|---|
| Instagram | Kazakhstan、Uzbekistan、Kyrgyzstan | 实车场景图、短视频、前后对比 | 发 Reels + 商品帖，引导私聊 | 简介里写城市、付款、配送 |
| TikTok | Kazakhstan、Uzbekistan 可测 | 15-30 秒安装/使用视频 | 小额测试素材，不先大投广告 | 投放国家和账号限制要执行前复核 |
| Facebook | Tajikistan、Kyrgyzstan 可观察 | 群组、Marketplace、图文帖 | 用本地群/Marketplace 测询盘 | 群规则复杂，不要刷屏 |

适合车内用品的视频结构：

```text
痛点 2 秒：手机放哪都不稳
安装 5 秒：支架怎么装
使用 8 秒：开车/导航/拿取
信任 3 秒：Almaty 现货/可配送
行动 2 秒：私信下单
```

社媒 P0 不看粉丝数，先看：

| 指标 | 说明 |
|---|---|
| 私聊数 | 是否有人真的问 |
| 有效询盘数 | 是否问价格、城市、配送 |
| 报价后继续聊 | 价格是否能接受 |
| 付款数 | 最终是否愿意付钱 |

### P0 承接工具：Telegram / WhatsApp

Telegram 和 WhatsApp 不是流量平台的替代品，而是订单承接工具。客户从 OLX、Instagram、TikTok 进来后，需要在聊天里完成信任建立和付款引导。

| 工具 | 适合用法 | 优点 | 缺点 |
|---|---|---|---|
| Telegram | Kazakhstan、Uzbekistan、Kyrgyzstan、Tajikistan 都可用 | 本地使用习惯强，适合发图、视频、位置、付款说明 | 聊天证据要整理，团队协作容易乱 |
| WhatsApp | 适合一对一客服和订单确认 | 商务感更强，客户容易保存联系方式 | 需要规范话术和标签 |

聊天里必须问清楚：

1. 客户在哪个城市。
2. 要哪个 SKU/颜色/数量。
3. 是否接受配送费。
4. 是否接受付款方式。
5. 是否接受售后规则。

聊天里必须发清楚：

```text
商品价格
配送费
预计发货时间
付款方式
退换/补发规则
```

小白团队最容易犯的错：

| 错误 | 后果 |
|---|---|
| 所有人都用自己手机接单 | 订单证据分散，无法复盘 |
| 不记录客户来源 | 不知道哪个渠道有效 |
| 不设置统一报价 | 客服乱报价 |
| 不提前说售后规则 | 交付后争议大 |

### P1 平台：Kaspi

Kaspi 是 Kazakhstan 最重要的电商/支付生态之一。它适合 P1，不适合作为 P0 的前置条件。

| 项 | 说明 |
|---|---|
| 国家 | Kazakhstan |
| 类型 | Marketplace + 支付/金融生态 |
| 适合阶段 | P1/P2 |
| 适合商品 | 已在 P0 证明有人买的 SKU，例如车载支架、收纳、手机周边 |
| 核心价值 | 用户信任强，平台内购买心智成熟 |
| 主要门槛 | Kaspi Pay、本地经营主体/账户、平台条件、可能的销售记录要求 |
| 小白建议 | 不要等 Kaspi 入驻完成才开始 P0；先用 OLX/社媒跑订单，再准备 Kaspi |

Kaspi 的正确位置：

```text
P0：不用 Kaspi 店铺也能先测需求
P1：用成交 SKU 准备 Kaspi 入驻
P2：做平台排名、评价、库存、价格运营
```

### P1 平台：Uzum

Uzum 是 Uzbekistan 的核心电商平台和生态之一。它适合第二市场和 P1/P2。

| 项 | 说明 |
|---|---|
| 国家 | Uzbekistan |
| 类型 | Marketplace + 金融/支付生态 |
| 适合阶段 | P1/P2 |
| 适合商品 | 手机周边、车内用品、轻小件、标准化商品 |
| 核心价值 | 增长快，用户线上购物习惯在形成 |
| 主要门槛 | 乌兹本地主体/个体/自雇、注册材料、税务配置、平台规则 |
| 小白建议 | 如果你们在 Tashkent 有收款和轻仓资源，Uzbekistan 可以反超 Kazakhstan 做实操首站 |

Uzum 不适合直接做 P0 的原因：

- 入驻资料需要准备。
- 本地主体和税务配置会拖慢第一单。
- 没有真实成交 SKU 时，盲目上架会变成铺货，不是验证。

### P1 平台：Olcha / Flip / Satu / Asaxiy

这些是本地平台或零售/平台型渠道，适合作为 P1 之后的补充，不适合作为第一天动作。

| 平台 | 国家 | 当前判断 | 怎么用 |
|---|---|---|---|
| Olcha | Uzbekistan | 可作为 Uzum 之外的 P1 平台 | P0 有成交 SKU 后，再准备入驻/合作 |
| Flip | Kazakhstan | 可作为 Kaspi 之外的 P1 补充 | 适合测试非 Kaspi 流量 |
| Satu | Kazakhstan | 信息需要进一步确认 | 先当线索，不放 critical path |
| Asaxiy | Uzbekistan | 更像零售/合作线索，第三方入驻公开信息不足 | P2 再联系合作 |

这里的关键判断：

```text
这些平台不是没价值，而是不适合作为第一单之前的必经步骤。
```

### P1/P2 平台：Wildberries / Ozon

Wildberries 和 Ozon 更适合做跨国复制和规模化，不适合一开始做 P0。

| 平台 | 覆盖 | 适合阶段 | 优点 | 风险 |
|---|---|---|---|---|
| Wildberries | Kazakhstan、Kyrgyzstan 等；Tajikistan 买家端有入场线索 | P1/P2 | 流量大，可跨市场 | 主体、结算、仓配、退货、平台规则重 |
| Ozon | Kazakhstan、Kyrgyzstan、Uzbekistan 等 | P1/P2 | 商品搜索和跨境能力强 | 本地主体/资料/结算/履约要逐国确认 |

适合上 WB/Ozon 的前提：

1. P0 已有真实付款订单。
2. SKU 售后低。
3. 有稳定补货。
4. 能处理退货。
5. 能接受平台规则和结算周期。

不适合：

- 还没确认用户要不要买。
- SKU 还在乱试。
- 没本地客服。
- 退货成本没算。

### 独立站

独立站不是 P0 的主战场。

| 用途 | 适合阶段 | 说明 |
|---|---|---|
| 商品说明页 | P1 | 放 FAQ、尺寸、安装视频、售后规则 |
| 复购入口 | P1/P2 | 已成交客户二次购买 |
| 品牌资产 | P2 | 做长期品牌和 SEO |
| 批发询盘 | P2/P3 | 给本地小卖家/分销商看目录 |

为什么不建议 P0 先做独立站：

- 中亚支付接入不一定顺。
- 陌生客户信任弱。
- 广告导到独立站可能只买来点击，不买来订单。
- 小团队会把时间花在装修网站，而不是验证订单。

P0 如果一定要用独立站，只做一页：

```text
5 个 SKU
真实图片和视频
城市和配送说明
WhatsApp/Telegram 按钮
付款说明
售后规则
```

### 分销/批发渠道

分销/批发是 P3，不是 P0。

| 渠道 | 对象 | 适合什么时候做 |
|---|---|---|
| 车品店 | 本地汽车用品门店 | C 端已有成交 SKU 后 |
| 小卖家 | OLX/Instagram/Marketplace 卖家 | P1 后，能提供稳定供货 |
| 本地进口商 | 有清关和仓储能力的贸易商 | P2/P3 |
| 批发市场 | 当地线下批发/零售档口 | 有样品、报价、交期后 |

P3 要准备：

- 批发价。
- 最小起订量。
- 样品。
- 交期。
- 坏件补发规则。
- 付款规则。
- 是否能贴标/换包装。

### 推荐渠道组合

#### Kazakhstan / Almaty 默认组合

```text
OLX.kz 发商品
+ Instagram 发短视频
+ Telegram/WhatsApp 私聊承接
+ Kaspi Pay/payment link/银行转账收款
+ Almaty 本地派送/自提
```

P1 再加：

```text
Kaspi
Flip
Wildberries/Ozon 评估
```

#### Uzbekistan / Tashkent 备选组合

```text
OLX Uzbekistan 发商品
+ Telegram/Instagram 引流
+ Payme/Click/Freedom Pay/本地转账收款
+ Tashkent 本地派送/自提
```

P1 再加：

```text
Uzum
Olcha
Ozon Uzbekistan 评估
```

#### Kyrgyzstan / Bishkek 小样组合

```text
Lalafo 发商品
+ Instagram/Telegram 私聊
+ Freedom Pay KG/本地转账
+ Bishkek 本地派送/自提
```

### 渠道选择的判断表

| 当前条件 | 应该选 |
|---|---|
| 还没有本地主体 | Classifieds + 社媒 + 聊天 |
| 有本地收款但没有平台店铺 | Classifieds + 社媒先跑 P0 |
| 已有 P0 成交 SKU | 准备 marketplace 入驻 |
| 有稳定库存和客服 | 上 Kaspi/Uzum/Olcha/Flip |
| 有退货处理能力 | 再看 Wildberries/Ozon |
| 需要沉淀品牌和 FAQ | 做一页独立站 |
| 有本地小卖家主动问批发 | 进入 P3 分销/批发 |

## 为什么 P0 不先做 Kaspi/Uzum

Kaspi 和 Uzum 是好渠道，但不一定适合“第一单之前”。

| 平台 | 问题 |
|---|---|
| Kaspi | 需要 Kaspi Pay、本地经营主体/账户、近期收款记录等条件 |
| Uzum | 需要乌兹注册主体/个体/自雇、注册材料、税务配置 |
| Wildberries/Ozon | 本地主体、税号、银行账户、仓配、退货、规则都更重 |

所以 P0 的合理路径是：

```text
先用 classifieds / 社媒找到陌生客户
-> 私聊确认商品和配送
-> 本地 payment link / 转账付款
-> 本地节点发货
-> 记录证据
```

## 名词梗概

### MVP

MVP = 最小可行版本。  
在这里指：不用做完整公司系统，只做能验证“陌生客户会付款、货能送到”的最小链路。

具体例子：

```text
1 个国家 + 1 个城市 + 5 个 SKU + 2 个渠道 + 1 个收款方式 + 1 个发货节点
```

### P0 / P1 / P2 / P3

| 名词 | 意思 |
|---|---|
| P0 | 第一优先级：完成首批真实陌生客户订单 |
| P1 | 第二优先级：把 P0 闭环复制成 SOP |
| P2 | 第三优先级：稳定出单和稳定利润 |
| P3 | 第四优先级：接触分销/批发客户 |

### 小宗商品

小宗商品 = 单件价值、重量、体积都比较小的商品。  
这里优先指车内小件、支架、收纳件、手机周边。

### 大宗商品

大宗商品 = 大批量、货值高、贸易链条重的商品。  
这里暂时不做，因为它更依赖清关、账期、合同、仓储、资金周转和 B2B 信用。

### SKU

SKU = 一个具体可售商品单位。  
颜色、型号、尺寸不同，通常算不同 SKU。

具体例子：

```text
黑色出风口车载支架 = 1 个 SKU
银色出风口车载支架 = 另 1 个 SKU
```

### D2C

D2C = 直接卖给消费者。  
这里指我们直接把车载支架卖给当地车主。

### B2B

B2B = 卖给商家。  
这里指卖给当地车品店、小卖家、分销商。

### Marketplace

Marketplace = 有买家流量和平台交易规则的电商平台。  
这里包括 Kaspi、Uzum、Wildberries、Ozon、Olcha。

### Classifieds

Classifieds = 分类信息平台。  
用户发商品信息，买家私聊或电话联系，平台通常不负责完整交易托管。

具体例子：

```text
OLX.kz、Lalafo、Somon
```

### Payment link

Payment link = 支付链接。  
卖家生成一个付款链接，客户打开链接付款。

### PSP

PSP = Payment Service Provider，支付服务商。  
它负责在线收款、交易处理、对账等能力。

具体例子：

```text
Payme、Click、Freedom Pay、Halyk EPAY
```

### Acquiring

Acquiring = 商户收单。  
它让商家可以收银行卡、钱包、二维码、线上付款。

### COD

COD = Cash on Delivery，货到付款。  
P0 不把全额 COD 算成功，因为客户没有提前承担付款风险，拒收概率高。

### Prepayment

Prepayment = 预付款。  
P0 最好用全额预付款；最低也要订金 + 明确尾款规则。

### 3PL

3PL = 第三方物流服务商。  
它负责仓储、发货、派送、退件等部分能力。

### Light warehouse

Light warehouse = 轻仓。  
这里指在目标国放少量库存，不是完整大仓。

具体例子：

```text
先在 Almaty 放 50-100 件小商品，由合作方发货
```

### Last-mile delivery

Last-mile delivery = 末端配送。  
指商品进入目标城市后，送到客户手里的那一段。

### Reverse logistics

Reverse logistics = 逆向物流。  
指退货、换货、补发、维修这类售后物流。

### HS code

HS code = 商品海关编码。  
它影响税率、监管条件、申报名称和清关材料。

### Customs clearance

Customs clearance = 清关。  
指货物进目标国时，海关检查、申报、征税、放行的过程。

### De minimis / duty-free threshold

De minimis = 小额免税或低值简化门槛。  
在这里主要看个人小包金额和重量限制。

当前调研里：

| 国家 | 小包相关门槛 |
|---|---|
| Kazakhstan | 个人小包约 €200 / 31kg |
| Kyrgyzstan | EAEU 口径约 €200 |
| Uzbekistan | Courier $200/月/收件人；postal $100 |
| Tajikistan | 未找到可靠公开个人小包限额 |
| Turkmenistan | 未找到可靠公开电商小包限额 |

### DAP / DDU / DDP

| 名词 | 意思 | MVP 处理 |
|---|---|---|
| DAP | 卖家送到指定地点，进口税费通常由买家/收件方处理 | 可用，但要讲清楚税费 |
| DDU | 未完税交付，税费不包含在卖家报价里 | P0 可用，但不要承诺包税 |
| DDP | 完税交付，卖家承担进口税费和清关责任 | P0 不建议承诺 |

### EAEU

EAEU = Eurasian Economic Union，欧亚经济联盟。  
在本项目里，Kazakhstan 和 Kyrgyzstan 受 EAEU 规则影响。

### EAC

EAC = EAEU 相关合规标志。  
电子、电气、部分消费品进入 EAEU 市场时可能需要对应认证或声明。

### TR CU / TR EAEU

TR CU / TR EAEU = 技术法规。  
它规定某些商品进入 EAEU 市场前需要满足什么安全、EMC、材料或限制要求。

### EMC

EMC = 电磁兼容。  
带电路板、充电器、无线设备等可能涉及 EMC 要求。

### LVD / Low voltage

LVD = 低压电气安全要求。  
充电器、电源适配器、小家电这类商品可能触发。

### RoHS

RoHS = 有害物质限制。  
电子电气产品可能涉及材料限制。

### RF

RF = Radio Frequency，无线射频。  
蓝牙、Wi-Fi、GPS、SIM、FM、遥控设备都可能触发无线相关审批。

### UN38.3

UN38.3 = 锂电池运输测试要求。  
带锂电池产品运输前通常需要相关测试文件。

### MSDS

MSDS = Material Safety Data Sheet，材料安全数据表。  
电池、液体、化学品、清洁剂、胶水等可能需要。

### IATA DGR

IATA DGR = 国际航空运输协会危险品规则。  
锂电池、液体、化学品等空运时可能受它约束。

### GMV

GMV = 商品交易总额。  
它是成交额，不等于利润。

### TPV

TPV = Total Payment Volume，支付交易总额。  
常用于支付平台或金融科技公司。

### AOV

AOV = Average Order Value，平均客单价。  
计算方式：

```text
总销售额 / 订单数
```

### Gross margin

Gross margin = 毛利。  
计算方式：

```text
售价 - 商品成本 - 头程/尾程物流 - 包装 - 平台费 - 支付手续费 - 补发退款成本
```

P0 可以不赚钱，但必须记录真实毛利。

### CAC

CAC = Customer Acquisition Cost，获客成本。  
计算方式：

```text
广告费 / 付款客户数
```

### Lead

Lead = 潜在客户线索。  
比如一个客户私聊问价格，但还没付款。

### Qualified lead

Qualified lead = 有明确购买意向的线索。  
比如客户已确认城市、款式、价格和付款方式，只差付款。

## 核心风险

| 风险 | 为什么危险 | 怎么处理 |
|---|---|---|
| 把 COD 当成功 | 客户没提前付款，拒收风险高 | P0 只认预付款/线上付款/转账 |
| 一开始上复杂电子 | 合规、运输、售后、兼容问题叠加 | P0 只做机械小件 |
| 五国同时做 | 每国支付、语言、履约、平台都不同 | 先 1 国 1 城市 |
| 只看市场不看本地资源 | 没收款/发货就无法闭环 | 先填上下文采集表 |
| 只做调研不做订单证据 | 没法判断市场是否真的成立 | 每单留聊天、付款、物流、售后证据 |
