# 从 0 开始的中亚电商 MVP 具体执行报告

前提：没有更多你们上下文时，先按这版执行。  
目标：完成 P0，拿到陌生真实客户主动付款订单，并交付成功。  
执行等级：🟡 可用。不是生产级跨境贸易体系。

## 0. 最小可行性版本

| 项 | 选择 |
|---|---|
| 国家 | Kazakhstan |
| 城市 | Almaty |
| 商品数量 | 5 个 SKU |
| 首批库存 | 每 SKU 10-20 件 |
| 渠道 | OLX.kz + Instagram + Telegram/WhatsApp |
| 付款 | 本地合作方 Kaspi Pay/payment link/银行转账 |
| 履约 | 中国小批量发到 Almaty 本地节点，再本地派送/自提 |
| 成功标准 | 1 个陌生客户完成预付款/线上付款，商品交付，证据完整 |
| 不追求 | 不追求利润、不追求五国铺开、不追求平台入驻 |

## 1. 先准备什么

### 1.1 团队角色

| 角色 | 负责什么 | 最低要求 |
|---|---|---|
| 项目负责人 | 决策国家、SKU、预算、复盘 | 每天看数据 |
| 供应链负责人 | 拿货、质检、包装、发货到本地节点 | 每 SKU 能稳定补货 |
| 本地负责人 | 收款、轻仓、派送、退货地址 | 在 Almaty 可执行 |
| 客服负责人 | 俄语/当地语私聊、报价、催付、售后 | 能每天回复 |
| 内容负责人 | 拍图、短视频、商品文案、上架 | 能产出真实素材 |
| 数据负责人 | 订单证据、渠道数据、复盘表 | 每单留证 |

一个人可以兼多个角色，但不能没人负责。

### 1.2 本地能力

P0 开始前必须拿到：

| 能力 | 最低标准 |
|---|---|
| 收款 | 能让客户用本地熟悉方式付款，并截图/对账 |
| 发货 | Almaty 本地能派送或约自提 |
| 退货/补发 | 有本地地址或明确“不退货，只补发/退款”的规则 |
| 客服 | 至少能用俄语回复价格、配送、付款、售后 |
| 对账 | 每笔订单能对应客户、SKU、金额、发货状态 |

没有本地收款，就不要投广告。

### 1.3 选择 5 个 SKU

先做这 5 类：

| SKU | 数量 | 为什么 |
|---|---:|---|
| 出风口车载手机支架 | 10-20 | 场景直观，视频好拍 |
| 仪表台/吸盘车载支架 | 10-20 | 和出风口支架形成对比 |
| 座椅缝隙收纳盒 | 10-20 | 低合规，车主需求明确 |
| 理线夹/挂钩组合 | 20-50 | 便宜、轻、小，可做加购 |
| 防滑垫/临时停车牌 | 20-50 | 低售后，适合套装 |

不做：

- 蓝牙。
- 无线充。
- 车充。
- 充电宝。
- 小家电。
- 香薰液。
- 功能性汽配。

### 1.4 内容素材

每个 SKU 必须准备：

| 素材 | 数量 | 说明 |
|---|---:|---|
| 白底图 | 1 | 看清商品 |
| 实车场景图 | 3 | 放在车里真实展示 |
| 尺寸图 | 1 | 降低尺寸误解 |
| 15-30 秒视频 | 1 | 展示安装、稳定性、使用前后 |
| 俄语标题 | 1 | 不要只用中文/英文 |
| 俄语卖点 | 3 | 直接讲功能、尺寸、配送 |

## 2. 第 1 周：把交易链路搭起来

### Day 1：定国家和城市

默认：

```text
Kazakhstan / Almaty
```

如果填表后发现 Uzbekistan 本地资源明显更强，就改成：

```text
Uzbekistan / Tashkent
```

不要五国同时做。

### Day 2：确认本地收款

Kazakhstan 优先顺序：

1. Kaspi Pay/payment link。
2. Halyk EPAY。
3. 本地银行卡/银行转账。
4. 合作方代收。

测试动作：

```text
内部转一笔小额测试款
保存付款截图
确认到账时间
确认手续费
确认能否退款
```

不通过就暂停广告和上架。

### Day 3：确认本地履约

必须跑一次内部测试：

```text
本地节点 -> Almaty 市内地址
```

记录：

- 发货时间。
- 到达时间。
- 运费。
- 是否能签收证明。
- 丢件/破损赔付规则。
- 退件地址。

### Day 4：确认商品和价格

每个 SKU 算一遍：

```text
售价
- 商品成本
- 中国到本地节点成本
- 本地配送
- 包装
- 支付手续费
- 预计补发/退款成本
= 单笔真实毛利
```

P0 可以不赚钱，但不能不知道亏在哪里。

### Day 5-7：准备内容和商品页

每个 SKU 写一个商品页模板：

```text
商品名：
价格：
颜色/规格：
适合场景：
尺寸：
材质：
配送城市：
付款方式：
发货时间：
售后规则：
```

## 3. 第 2 周：上架和获客

### OLX.kz

动作：

1. 每个 SKU 发 1 条商品。
2. 标题用俄语。
3. 图片必须是真实场景图。
4. 描述写清楚价格、城市、付款、配送。
5. 每天刷新/优化一次标题和首图。

目标：

```text
5 个 SKU -> 5 条 OLX 商品
```

### Instagram

动作：

1. 建一个简单账号。
2. 发 5 个商品帖。
3. 发 5 条 Reels/短视频。
4. Bio 写清楚城市、付款、配送、联系方式。
5. 私聊统一导向 WhatsApp/Telegram。

目标：

```text
5 个商品帖 + 5 条短视频
```

### Telegram / WhatsApp

动作：

1. 准备统一客服话术。
2. 每个询盘记录来源。
3. 报价后必须追问城市和配送方式。
4. 付款前必须发送售后规则。

不要群发刷屏。

## 4. 第 3 周：拿首单

### 客服流程

```text
客户问价
-> 确认城市
-> 确认款式
-> 发实拍图/视频
-> 说明价格和配送费
-> 说明付款方式
-> 说明售后规则
-> 发 payment link/转账信息
-> 客户付款
-> 生成订单编号
-> 本地发货
-> 保存交付证据
```

### 付款标准

算 P0：

- 平台内付款。
- Payment link。
- 本地转账。
- 全额预付款。

不算 P0：

- 全额 COD。
- 口头预订。
- 朋友帮买。
- 合作方自己买。
- 只加购物车或只问价。

### 订单编号

格式：

```text
CA-KZ-YYYYMMDD-CHANNEL-SKU-001
```

例子：

```text
CA-KZ-20260616-OLX-CARHOLDER-001
```

## 5. 数据怎么记

每条渠道每天记：

| 指标 | 说明 |
|---|---|
| 浏览/曝光 | 多少人看到 |
| 咨询 | 多少人私聊 |
| 有效咨询 | 问了城市/价格/配送的人 |
| 报价后回复 | 报价后还继续聊的人 |
| 付款 | 真正付款的人 |
| 放弃原因 | 太贵、不能付款、配送慢、不信任、没现货 |

每个订单必须留：

- 商品页链接/截图。
- 聊天截图。
- 付款截图/交易号。
- 发货证明。
- 签收/交付证明。
- 售后记录。

## 6. 什么时候算成功

P0 成功：

```text
1 个陌生客户
+ 主动从渠道进入
+ 完成预付款/线上付款
+ 商品交付完成
+ 证据完整
```

P1 启动：

```text
同一国家同一链路完成 10 单
或 2 个 SKU 各完成 3 单
```

## 7. 如果 7 天没有首单

按问题拆：

| 现象 | 判断 | 动作 |
|---|---|---|
| 没曝光 | 渠道/标题/图片问题 | 换标题、换首图、加短视频 |
| 有曝光无咨询 | 商品/价格/图片问题 | 换 SKU 或做套装 |
| 有咨询无付款 | 信任/付款/配送问题 | 增加实拍、明确本地库存、换付款方式 |
| 有付款但发不出 | 履约问题 | 停止获客，先修履约 |
| 售后多 | SKU 问题 | 淘汰 SKU |

## 8. 最小预算

不含团队人力。

| 项 | 建议 |
|---|---:|
| 样品/首批库存 | 每 SKU 10-20 件 |
| 本地节点/轻仓 | 先谈按件/按周，不签长期 |
| 素材 | 自拍优先 |
| 广告 | 第一周可 0；收款/履约跑通后再小额 |
| 补发/退款预留 | 按销售额 10%-20% 预留 |

## 9. 付费广告什么时候开

只有同时满足才开：

1. 本地收款已测通。
2. 本地发货已测通。
3. 商品素材齐。
4. 售后规则明确。
5. 客服能及时回复。

小额广告测试：

```text
每天小预算
只投 2 个最容易懂的 SKU
只投 Almaty
只导向 WhatsApp/Telegram
只看付款，不只看点击
```

## 10. 下一步进入 P1

P0 跑通后，不要马上扩五国。先做：

1. 保留成交 SKU。
2. 淘汰高咨询低付款 SKU。
3. 固化最有效话术。
4. 固化收款和对账方式。
5. 固化本地发货方式。
6. 开始准备 Kaspi 入驻材料。
7. 复制到 Uzbekistan/Tashkent 做第二条链路。

## 11. 这版执行报告的边界

这版不解决：

- 本地主体注册细节。
- 税务结构。
- 每个 SKU 的正式 HS code 和认证结论。
- 大规模仓储。
- B2B 批发账期。
- 品牌站长期运营。

这些等 P0/P1 真实数据回来后再做。
