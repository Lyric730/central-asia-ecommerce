# 从0开始的中亚电商 MVP 具体执行报告

版本：v2
更新日期：2026-06-20
默认目标市场：Kazakhstan / Almaty
默认阶段目标：P0 完成首批真实订单，保证交易链路闭环

## 0. 这份文档到底要解决什么

这不是调研报告，也不是商业计划书。它是一份给小白团队照着执行的 MVP SOP。

本项目的第一阶段不要追求完整公司、完整平台、完整跨境系统。第一阶段只验证一件事：

```text
一个陌生客户
从一个真实渠道看到商品
主动咨询
接受报价和交付规则
完成预付款或线上付款
收到商品
全过程证据完整
```

只要这件事没发生，市场规模、平台 GMV、供应链资源、资金充足都还只是背景信息。

## 1. 当前默认方案

| 模块 | 默认选择 | 为什么 |
|---|---|---|
| 国家 | Kazakhstan | 电商、支付、物流、购买力和平台生态综合最好 |
| 城市 | Almaty | 最大城市，经济、金融、交通和中资资源更集中 |
| P0 商品方向 | 车内用品 + 无电池手机周边 | 轻小、好展示、低售后、低认证风险 |
| P0 商品数量 | 5 个 SKU | 够测试，不会把客服和库存搞乱 |
| P0 首批库存 | 每个 SKU 10-20 件 | 有现货感，又不压太多货 |
| P0 获客渠道 | OLX.kz + Instagram + Telegram/WhatsApp | 不依赖复杂平台入驻，能最快拿真实询盘 |
| P1 主平台 | Kaspi Marketplace | 哈萨克斯坦核心电商/支付生态，但入驻前置条件更重 |
| P0 收款 | 本地合作方收款、Kaspi Pay、Halyk EPAY、银行转账，按实际可用性排序 | P0 先解决能不能收钱，不追求最优结构 |
| P0 履约 | 中国小批量到 Almaty 本地节点，再本地派送或自提 | 降低跨境单票不确定性 |
| P0 成功标准 | 至少 1 个陌生客户完成付款、交付和证据留存 | 不把口头意向、朋友帮买、货到付款当成功 |

## 2. 关键修正：Kaspi 不是 P0 前置条件

Kaspi 很重要，但不能把它放在第一单之前。

我复核了 Kaspi 官方合作伙伴指南。成为 Kaspi.kz 商店合作伙伴，需要先接入 Kaspi Pay，并且在最近 2 个月内至少 20 天通过 Kaspi Pay 收款。官方说明还写明，企业需要销售实物商品，服务类应去“Объявления”服务。也就是说，如果你们现在没有本地主体、Kaspi Pay 和收款记录，Kaspi 不能作为 P0 的唯一入口。

正确排序：

| 阶段 | Kaspi 的位置 | 动作 |
|---|---|---|
| P0 | 研究和准备，不作为唯一入口 | 跑 OLX/社媒订单，同时确认本地主体、Kaspi Pay、收款记录要求 |
| P1 | 入驻任务 | 用已成交 SKU 去申请或通过合作方上架 |
| P2 | 主平台运营 | 做评价、排名、库存周转、售后、复购 |

## 3. P0 的硬边界

### 3.1 P0 算成功的标准

同时满足以下条件才算 P0 成功：

| 条件 | 标准 |
|---|---|
| 客户来源 | 陌生客户，来自 OLX、Instagram、Telegram、WhatsApp、线下转介绍等真实获客渠道 |
| 付款 | 预付款、全款线上支付、payment link、银行转账、本地电子钱包转账之一 |
| 交付 | 商品实际交给客户，或者客户明确确认收货 |
| 证据 | 有商品页截图、聊天截图、付款凭证、发货凭证、收货/交付凭证 |
| 复盘 | 订单记录能回溯 SKU、渠道、价格、币种、汇率、物流、售后 |

### 3.2 不算 P0 成功的情况

| 情况 | 为什么不算 |
|---|---|
| 朋友帮买 | 不是陌生客户需求 |
| 合作方自己买 | 不能证明市场需求 |
| 只加购物车 | 没有付款 |
| 只问价 | 没有交易 |
| 全额 COD | 客户没有提前承担付款风险，拒收风险没有验证 |
| 免费送样 | 没有验证支付意愿 |
| 只拿到批发口头意向 | 不是 P0 的 C 端闭环 |

## 4. 团队角色和责任

一个人可以兼多个角色，但每个责任必须有人负责。

| 角色 | 负责什么 | 每日交付物 |
|---|---|---|
| 项目负责人 | 阶段目标、SKU 取舍、预算、复盘、推进卡点 | 每日复盘结论和次日任务 |
| 供应链负责人 | 拿货、质检、包装、发货到本地节点 | SKU 成本表、质检记录、发货记录 |
| 本地负责人 | 收款、轻仓、本地派送、自提、退换货地址 | 收款截图、库存台账、本地物流记录 |
| 客服负责人 | 俄语/当地语言承接、报价、催付、售后 | 聊天记录、客户问题表、付款转化表 |
| 内容负责人 | 图片、短视频、商品标题、商品描述、上架 | 每个 SKU 的图文素材包 |
| 数据负责人 | 订单台账、渠道数据、复盘指标 | 每日数据表和异常记录 |
| 合规负责人 | HS code、认证、禁售、平台规则、税费假设 | SKU 合规预筛表 |

最低配置：

```text
中国端 2 人：供应链 + 内容/数据
本地端 1 人：收款 + 轻仓 + 派送协调
客服 1 人：可由本地端兼任，但必须能稳定回复
```

## 5. 执行前 48 小时必须确认的事

这部分没确认，不要投广告，不要大量备货。

### 5.1 本地能力清单

| 能力 | 最低标准 | 验证方式 |
|---|---|---|
| 本地收款 | 至少一种客户愿意用的收款方式 | 内部小额测试，保存到账截图 |
| 本地派送 | Almaty 市内能派送或约自提 | 跑一次内部测试单，记录时效和费用 |
| 退换货地址 | 有可接收退件的本地地址 | 用内部包裹模拟退件 |
| 客服语言 | 至少俄语可回复价格、配送、付款、售后 | 写 20 条标准话术并试聊 |
| 库存保管 | 能保管 50-100 件小商品 | 拍轻仓照片，记录出入库 |
| 对账 | 每笔订单能对应付款和发货 | 做一笔内部测试订单 |

### 5.2 本地合作方尽调

如果 P0 要靠本地合作方收款或发货，必须问清：

| 问题 | 合格答案 |
|---|---|
| 能不能收本地客户付款 | 能，且能提供截图/流水/对账 |
| 款项多久能转给我们 | 明确 T+几天，手续费多少 |
| 能不能保管库存 | 能，库存谁负责丢损要写清 |
| 能不能发货或派送 | 能，列出派送范围、费用、时效 |
| 退货怎么处理 | 有地址、有拍照、有责任划分 |
| 是否能配合 Kaspi Pay/Kaspi 店铺 | 能说明主体、账号、权限、风险 |
| 合作终止怎么办 | 库存、客户、账号、资金怎么交接 |

红线：

```text
不能看流水，不合作。
不能对账，不合作。
不愿写责任边界，不合作。
要求大额预付款，不合作。
账号、客户、库存都只在对方手里，不合作。
```

## 6. SKU 选择 SOP

### 6.1 P0 SKU 原则

P0 只选能快速验证交易链路的商品，不选“看起来市场很大但售后复杂”的商品。

必须满足：

| 标准 | 要求 |
|---|---|
| 轻小 | 单件最好小于 500g |
| 低损耗 | 不易碎、不漏液、不变质 |
| 低合规 | 无电池、无蓝牙、无 Wi-Fi、无射频、无插电 |
| 易展示 | 一张场景图或 15 秒视频能讲清价值 |
| 易替换 | 坏了可以补发，不需要维修 |
| 通用性 | 不强依赖车型、系统、语言、尺码 |
| 低客诉 | 不涉及安全、医疗、功效承诺 |

### 6.2 默认首批 5 个 SKU

| SKU | 首批数量 | 核心卖点 | 主要风险 | P0 处理 |
|---|---:|---|---|---|
| 出风口车载手机支架 | 10-20 | 导航稳定、安装简单 | 兼容部分出风口差 | 商品页写清适配类型 |
| 仪表台/吸盘手机支架 | 10-20 | 不占出风口、视角灵活 | 吸盘稳定性 | 实拍颠簸测试视频 |
| 座椅缝隙收纳盒 | 10-20 | 车内整洁、防掉落 | 车型缝隙差异 | 标注尺寸，提醒测量 |
| 理线夹/挂钩组合 | 20-50 | 便宜、轻小、可加购 | 客单低 | 做套装，不单独卖 |
| 防滑垫/临时停车牌 | 20-50 | 低价、易理解 | 同质化 | 用真实车内场景图提高信任 |

### 6.3 明确暂缓的 SKU

| SKU | 暂缓原因 | 什么时候再看 |
|---|---|---|
| 充电宝 | 电池运输、UN38.3、MSDS、售后和安全责任 | P1 后，有合规文件和退货能力 |
| TWS 耳机 | 蓝牙、电池、质量波动、退换货高 | P1/P2，有稳定供应商和 EAC 路径 |
| 车载充电器 | 发热、电气安全、认证、客诉风险 | P1 后，拿到合规资料 |
| 无线 CarPlay | 兼容性复杂、固件问题、退货率风险 | P2 后，有技术客服 |
| 智能手环/手表 | App、语言、蓝牙、电池、售后 | P2 后 |
| 小家电 | 插头、电压、重量、认证、破损 | P2 后 |
| 香薰液体 | 液体运输、泄漏、气味主观 | 暂不做 |
| 功能性汽车配件 | 安全责任、车型匹配复杂 | 暂不做 |

### 6.4 SKU 打分表

每个候选 SKU 先打分，低于 70 分不进入 P0。

| 指标 | 分值 | 评分说明 |
|---|---:|---|
| 需求直观 | 15 | 用户一眼能不能懂 |
| 展示容易 | 10 | 能不能用短视频讲清 |
| 轻小易发 | 15 | 重量、体积、破损风险 |
| 合规简单 | 20 | 是否避开电池、无线、插电、液体 |
| 售后简单 | 15 | 坏件能否低成本补发 |
| 供应稳定 | 10 | 是否能 7 天内补货 |
| 毛利空间 | 10 | 扣除物流后是否有空间 |
| 差异化 | 5 | 是否能靠套装、图文、场景做差异 |

## 7. 供应链和质检 SOP

### 7.1 拿货前问供应商

| 问题 | 为什么 |
|---|---|
| 是否有现货 | P0 不等长交期 |
| 最小起订量多少 | 控制首批库存 |
| 是否支持混批 | 方便测多个 SKU |
| 是否能提供中性包装 | 避免包装语言和品牌不适配 |
| 是否能补货 | P1 要复制 |
| 是否有质检报告/认证资料 | P1/P2 可能用到 |
| 是否支持坏件补发 | 降低售后成本 |
| 是否有同款多档质量 | 避免拿低质版本 |

### 7.2 到货质检

每个 SKU 抽检至少 20%，不足 10 件的全部检查。

| 检查项 | 通过标准 |
|---|---|
| 外观 | 无明显划痕、污渍、变形 |
| 功能 | 支架能夹住手机，收纳盒能正常安装 |
| 尺寸 | 和详情页一致 |
| 包装 | 不破损，不露中文批发标签 |
| 配件 | 螺丝、贴片、说明卡齐全 |
| 场景测试 | 能在真实车内安装并拍摄 |

质检后给每个 SKU 留：

```text
实物照片
包装照片
尺寸照片
安装视频
问题件照片
```

## 8. 报价和利润测算 SOP

P0 可以不赚钱，但不能不知道亏在哪里。

### 8.1 单 SKU 报价表

| 项目 | 公式或填写 |
|---|---|
| 中国采购价 | RMB |
| 中国国内运费 | RMB |
| 包装成本 | RMB |
| 中国到 Almaty 头程 | RMB/KZT |
| 本地仓储/人工 | KZT |
| Almaty 市内派送 | KZT |
| 支付手续费 | KZT |
| 平台/广告费 | P0 可先为 0 |
| 预留售后 | 建议销售额 10%-20% |
| 汇率缓冲 | 建议 5%-10% |
| 最低售价 | 全部成本 + 缓冲 |
| 测试售价 | 最低售价上浮 10%-30% |

### 8.2 报价规则

| 场景 | 规则 |
|---|---|
| 单件购买 | 商品价 + 配送费分开报 |
| 2 件以上 | 做套装价，提高客单 |
| 客户嫌贵 | 不马上降价，先问和哪个商品比 |
| 客户要求包邮 | 可以把配送费并入售价，但台账里要拆开 |
| 客户要求 COD | 只接受部分订金，或者不算 P0 成功 |

示例：

```text
商品：车载手机支架
售价：4,900 KZT
Almaty 配送：1,000 KZT
2 件套装：8,900 KZT
付款：Kaspi Pay / bank transfer
发货：今天或明天
售后：非人为损坏 7 天内补发一次
```

## 9. 内容和本地化 SOP

### 9.1 每个 SKU 必须准备的素材

| 素材 | 数量 | 标准 |
|---|---:|---|
| 白底图 | 1 | 看清商品 |
| 真实车内场景图 | 3 | 不要只用供应商图 |
| 尺寸图 | 1 | 用厘米标清 |
| 安装视频 | 1 | 15-30 秒 |
| 使用前后对比 | 1 | 展示痛点和效果 |
| 包装图 | 1 | 建立现货信任 |
| 俄语标题 | 3 个版本 | 用于 A/B 测试 |
| 俄语卖点 | 5 条 | 功能、尺寸、配送、售后 |

### 9.2 商品页模板

```text
商品名：
适用场景：
价格：
城市：
是否现货：
配送方式：
付款方式：
颜色/规格：
尺寸：
材质：
包含什么：
不适合什么：
售后规则：
联系方式：
```

### 9.3 俄语标题写法

标题不要写品牌愿景，要写用户搜索词。

示例：

```text
Автомобильный держатель для телефона, доставка по Алматы
Органайзер в щель сиденья авто, в наличии Алматы
Держатель телефона в машину, крепление на воздуховод
```

### 9.4 短视频脚本

```text
0-2 秒：展示痛点，比如手机放不稳、车内线乱
3-7 秒：安装商品
8-15 秒：真实使用，比如导航、收纳、拿取
16-20 秒：展示尺寸、材质、包装
21-25 秒：显示 Almaty 现货、可配送、私信下单
```

不要在视频里只放商品旋转图。P0 要证明真实场景能用。

## 10. 渠道执行 SOP

### 10.1 渠道组合

P0 不选单一平台，而是组合：

```text
OLX.kz 负责主动搜索流量
Instagram/TikTok 负责场景展示
Telegram/WhatsApp 负责咨询和成交
本地收款工具负责付款
本地轻仓负责交付
```

### 10.2 OLX.kz

OLX.kz 是 P0 的第一优先级渠道之一，因为它不要求先完成复杂 Marketplace 入驻。

执行步骤：

1. 注册账号并完成手机号验证。
2. 只上架 5 个 SKU，不要铺 50 个商品。
3. 每个 SKU 单独发帖，不要把所有商品塞进一个帖子。
4. 标题用俄语，包含商品词、车用场景、城市。
5. 第一张图用真实车内图。
6. 描述里写清价格、城市、配送、付款、联系方式。
7. 每天记录浏览、收藏、咨询、有效咨询、付款。
8. 3 天没有咨询，换首图和标题。

OLX 帖子结构：

```text
标题：
价格：
城市：Almaty
现货：有/无
配送：Almaty 市内可配送/可自提
付款：Kaspi Pay / bank transfer / payment link
商品说明：
适合：
不适合：
售后：
联系：WhatsApp / Telegram
```

注意：OLX 官方规则禁止重复或近似重复广告。不要用同一商品反复刷重复帖，避免被删帖或降权。

### 10.3 Instagram / TikTok

P0 用社媒不是为了做品牌，而是为了测试卖点。

第一周内容计划：

| 天数 | 内容 |
|---|---|
| Day 1 | 5 个 SKU 的实拍图帖 |
| Day 2 | 车载手机支架安装视频 |
| Day 3 | 收纳盒使用前后对比 |
| Day 4 | 理线夹/挂钩套装视频 |
| Day 5 | 防滑垫/停车牌低价套装 |
| Day 6 | 客户常见问题帖 |
| Day 7 | 现货和配送说明 |

Bio 必须写：

```text
Almaty
Car accessories / phone holders
Delivery / pickup
WhatsApp: xxx
Telegram: xxx
Payment: Kaspi / bank transfer
```

### 10.4 Telegram / WhatsApp

这两个工具是成交承接，不是发广告的地方。

客服标准流程：

```text
客户咨询
-> 询问城市
-> 确认 SKU/颜色/数量
-> 发送实拍图或视频
-> 报商品价和配送费
-> 说明付款方式
-> 说明发货时间
-> 说明售后规则
-> 发送付款信息
-> 客户付款
-> 生成订单号
-> 发货
-> 回传交付证据
-> 询问满意度/评价
```

客服必须记录：

| 字段 | 示例 |
|---|---|
| 来源 | OLX / Instagram / TikTok |
| 客户城市 | Almaty |
| 客户问题 | 价格、配送、是否现货 |
| 报价 | 4,900 KZT + 1,000 KZT 配送 |
| 是否付款 | 是/否 |
| 放弃原因 | 太贵/不信任/配送慢/付款不方便 |

## 11. 收款 SOP

### 11.1 收款优先级

| 优先级 | 方式 | 适用条件 | 风险 |
|---:|---|---|---|
| 1 | Kaspi Pay / Kaspi 转账 | 本地合作方或主体可用 | 对账和资金归属要写清 |
| 2 | Halyk EPAY / payment link | 有商户终端或本地商户支持 | 接入和结算资料更重 |
| 3 | 本地银行转账 | 客户接受转账 | 用户体验一般 |
| 4 | 订金 + 尾款 | 客户坚持 COD 时 | 不把全额 COD 计入 P0 成功 |
| 5 | 全额 COD | 只作为观察 | 不计入 P0 成功 |

Halyk EPAY 官方说明其提供面向商户的在线支付能力，并支持银行卡等付款方式。它适合 P1/P2 接入，不一定适合第一天就作为唯一收款方式。

### 11.2 收款测试

上线前必须完成一笔内部测试：

```text
测试金额：小额
付款人：团队成员或本地测试人
收款方式：真实 P0 计划使用方式
记录：付款截图、到账截图、手续费、到账时间、退款方式
```

如果收款测试失败：

```text
暂停上架和投放
先修收款
不要用“客户可以先下单，付款后面再说”的方式硬跑
```

## 13. 售后 SOP

P0 不要承诺复杂售后，只承诺能执行的规则。

建议规则：

```text
签收后 24 小时内发现非人为损坏，可补发一次或退款。
尺寸不适配但商品未损坏，不默认包退；可视情况换货。
客户下单前已明确尺寸和适用场景，因个人原因退货，配送费不退。
```

客服必须在付款前发售后规则，避免交付后争议。

## 16. 如果 7 天没有首单

不要直接说“市场不行”，按漏斗查。

| 现象 | 判断 | 动作 |

| 有浏览无私信 | 图文或价格问题 | 换场景图、加尺寸图、做套装 |

## 18. P1 到 P2 的门槛

P2 不是“订单多一点”，而是稳定出单且真实毛利为正。

P2 门槛：

| 指标 | 标准 |
|---|---|
| 时间 | 连续 30 天有订单 |
| 毛利 | 扣除采购、头程、本地配送、支付、广告、补发、退款后为正 |
| 售后 | 售后率可控，能定位问题 |
| 库存 | 不缺货，不大量积压 |
| 渠道 | 至少 2 个渠道能产生订单 |
| 客服 | 回复和对账不依赖单个人脑记 |

## 19. P3 分销/批发准备

P3 不要太早做，但从 P1 开始保留证据。

分销/批发客户会问：

| 问题 | 需要准备 |
|---|---|
| 批发价多少 | 阶梯报价 |
| MOQ 多少 | 最小起订量 |
| 交期多久 | 现货和补货周期 |
| 坏件怎么处理 | 补发/退货规则 |
| 能不能贴牌 | 包装和贴标能力 |
| 有没有销售证明 | C 端成交截图、评价、复购 |
| 能不能赊账 | P3 初期不建议赊账，至少订金 |

批发报价模板：

| 数量 | 单价 | 付款 | 交期 |
|---:|---:|---|---|
| 10-49 | 零售价 80% | 全款 | 现货 |
| 50-199 | 零售价 65%-70% | 50% 订金 + 50% 发货前 | 7-14 天 |
| 200+ | 单独报价 | 订金 + 尾款 | 14-30 天 |

## 20. 风险清单

| 风险 | 触发点 | 处理 |
|---|---|---|
| 收款不可控 | 款在合作方账户，无法对账 | 每日对账，合同写清 T+几天结算 |
| 库存不可控 | 货在本地节点但无台账 | 每日库存照片和出入库记录 |
| 价格失真 | 没算税费、物流、售后 | 单 SKU 成本表强制填写 |
| 合规误判 | 电子品类没有 EAC/电池资料 | P0 避开复杂 SKU |
| 客服失控 | 多人乱报价 | 统一报价表和话术 |
| 渠道误判 | 只看浏览，不看付款 | 只用付款和有效询盘判断 |
| 合作方风险 | 本地资源全在一个人手里 | 至少准备备选合作方 |
| 汇率风险 | KZT 波动影响毛利 | 报价预留 5%-10% 汇率缓冲 |
