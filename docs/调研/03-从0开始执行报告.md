# 从0开始的中亚电商 MVP 具体执行报告

版本：v2
更新日期：2026-06-20
默认目标市场：Kazakhstan / Almaty
默认阶段目标：P0 完成首批真实订单，保证交易链路闭环

## 0. 这份文档到底要解决什么

这不是调研报告，也不是商业计划书。它是一份给小白团队照着执行的 MVP SOP。

本项目的第一阶段不要追求完整公司、完整平台、完整跨境系统。第一阶段只验证一件事：

```text
一个陌生客户
从一个真实渠道看到商品
主动咨询
接受报价和交付规则
完成预付款或线上付款
收到商品
全过程证据完整
```

只要这件事没发生，市场规模、平台 GMV、供应链资源、资金充足都还只是背景信息。

| 模块 | 默认选择 | 为什么 |

| P0 首批库存 | 每个 SKU 10-20 件 | 有现货感，又不压太多货 |
| P0 获客渠道 | OLX.kz + Instagram + Telegram/WhatsApp | 不依赖复杂平台入驻，能最快拿真实询盘 |

| P0 履约 | 中国直发：邮政/ePacket、菜鸟、跨境专线、商业快递 | 不自己碰铁路整柜，先通过可追踪包裹服务发货 |
| P0 成功标准 | 至少 1 个陌生客户完成国际付款、中国发货、Kazakhstan 妥投/确认收货和证据留存 | P0-A 验证的是跨境直发闭环，不等于本地电商闭环 |

### 1.1 当前主线：P0-A 纯跨境直发

你们这次的前提是：初期不找本地人，不先做本地主体、本地仓、本地收款、本地派送。

这个前提可行，但要把目标说准：

```text
P0-A：跨境直发首单闭环
= 陌生客户在线付款
+ 中国端发出可追踪包裹
+ 包裹进入 Kazakhstan 并妥投或客户确认收货
+ 付款、发货、物流、售后证据可复盘
```

它不验证 Kaspi、本地仓配、货到付款、本地退货和本地复购。那些是 P0-B/P1 之后的事。

### 1.2 直接用人民币交易行不行

结论：不建议把人民币作为 Kazakhstan C 端客户看到的默认付款币种。

| 方案 | 能不能用 | 判断 |
|---|---|---|
| 客户直接付 RMB/CNY | 技术上可能，但不建议做 P0 默认方案 | 客户理解成本高，银行换汇、拒付、客服解释都会增加摩擦 |
| 客户用 Visa/Mastercard 付 USD/KZT | 推荐 | 客户只要能刷国际卡，换汇由发卡行和收单机构处理 |
| 客户用 PayPal 付款 | 可作为 P0 快速方案 | 适合支付链接、发票、自建站 checkout；但 Kazakhstan 用户习惯未必强 |
| 你们后台结算成人民币 | 可以 | 这是你们和 PSP/银行之间的结算问题，不应该让客户承担 |

执行口径：

```text
对客户报价：USD，必要时附 KZT 参考价。
对内核算：CNY 成本 + USD 收入 + 汇率缓冲。
不要在广告和聊天里主推“请付人民币”。
```

### 1.3 Visa/Mastercard 怎么接

Visa/Mastercard 不是你们直接去找卡组织开账户，而是通过 PSP 或支付平台接入。

PSP = 支付服务商，负责商户审核、卡支付通道、风控、拒付处理、结算。

P0 可选路径：

| 路径 | 怎么做 | 优点 | 缺点 | 推荐位置 |
|---|---|---|---|---|
| PayPal Business | 注册企业/商户账户，先用 PayPal invoice、PayPal.Me、付款链接或 Checkout | 启动最快，可无完整网站先收款 | Kazakhstan 用户可能不如本地支付习惯强；有风控和冻结风险 | P0 第一备选 |
| Airwallex / Oceanpayment / PingPong 等 PSP | 提交主体、网站、商品、物流、退款政策，开通卡收单 | 可直接收 Visa/Mastercard，体验更像正规独立站 | 审核资料更多，可能要求网站合规、交易风控 | P0/P1 并行申请 |
| Shopify/SHOPLINE/Shoplazza + 支付插件 | 建简单商品页，接 PayPal 或 PSP 插件 | 比纯聊天收款更可信，方便订单记录 | 月费和插件配置成本更高 | P0 可用，但别一开始做复杂站 |
| Wise/银行转账/国际汇款 | 发付款信息让客户转账 | 对 B2B 可行 | C 端极重，到账慢，客户信任弱 | P3 批发再考虑 |

最低资料清单：

| 资料 | 具体内容 |
|---|---|
| 主体资料 | 中国大陆公司或个体资料、法人/受益人 KYC、联系方式 |
| 网站/页面 | 商品页、价格、发货地、物流时效、退款政策、隐私政策、联系方式 |
| 商品资料 | SKU 图、规格、价格、是否含电池/液体/无线发射、质检方式 |
| 风控材料 | 发货凭证样例、物流追踪方式、售后处理规则、拒付处理负责人 |
| 结算设置 | 结算币种、提现账户、汇率损耗、手续费口径 |

P0 不要等所有 PSP 都审核完。正确做法：

```text
Day 1：PayPal Business / PayPal 发票或付款链接先跑。
Day 1-3：同时申请 1-2 个可收 Visa/Mastercard 的 PSP。
Day 4-7：谁先能真实收款，就用谁跑第一单。
```

### 1.4 能不能从中国直接发货

可以。初期不找本地人时，发货主线应该是中国直发包裹，不是本地派送。

可选方式：

| 方式 | 适合什么 | 优点 | 风险 |
|---|---|---|---|
| 中国邮政/ePacket/EMS 类服务 | 低客单、轻小件、非急件 | 覆盖广，成本相对低 | 时效波动，追踪节点可能慢 |
| 菜鸟/跨境电商物流 | 跨境电商小包、标准线 | 端到端体验更电商化，有标准/经济/优先产品 | Kazakhstan 具体线路和报价要逐票确认 |
| 中哈/中亚专线货代 | 轻小商品批量测试 | 可能更熟中亚清关和末端 | 服务质量差异大，要先发测试包 |
| DHL/UPS/FedEx 等商业快递 | 高客单、急件、样品 | 快、追踪强 | 成本高，客户可能因税费/派送费拒收 |

P0 商品选择要配合中国直发：

| 要求 | 原因 |
|---|---|
| 单件尽量低于 2kg | 小包可选项更多，运费更可控 |
| 单票申报价值尽量低于 EUR 200 | Kazakhstan 对个人邮包有 EUR 200/31kg 的免税阈值，但只适用于个人自用，不适用于商业进口 |
| 不碰电池、液体、磁性强、无线发射、医疗、儿童安全类 | 物流限制、认证和退货风险更高 |
| 不做高破损、高尺寸运费、强安装商品 | 首单闭环阶段不值得增加履约变量 |
| 如实申报品名、数量、价值 | 低报、错报会带来扣关、罚款、拒收和账号风控 |

客户话术必须透明：

```text
Ships from China.
Estimated delivery: 15-35 days depending on carrier and customs.
Import duties/taxes, if any, are paid by the buyer.
Tracking number will be provided after dispatch.
```

### 1.5 中欧班列能不能用

能用，但不能按“我们自己把第一单小包放上中欧班列”理解。

中欧班列是什么：国际铁路干线运输网络，主要服务集装箱、整票货、批量货和货代线路。

P0 的正确用法：

```text
你们不直接订铁路舱位。
你们找中国邮政、菜鸟、跨境小包、中哈专线、货代。
如果对方的线路底层用了铁路干线，那是物流商的路由选择。
你们只考核：价格、时效、能否追踪、丢损赔付、是否能到 Kazakhstan。
```

判断一个物流商能不能跑 P0，只问 8 个问题：

| 问题 | 通过标准 |
|---|---|
| 能不能从中国发 Kazakhstan C 端地址 | 能，且给出覆盖城市 |
| 是否支持单票小包 | 能发 1 件，不要求大批量 |
| 是否全程可追踪 | 至少有揽收、出境、入境、派送/妥投节点 |
| 是否接受车内用品/手机支架/内饰 | 明确可发，且列出禁运限制 |
| 预计时效 | 给出工作日/自然日范围，不只说“很快” |
| 费用结构 | 首重、续重、挂号费、偏远费、退件费 |
| 丢件/破损怎么赔 | 有书面赔付规则 |
| 客户拒收或清关失败怎么办 | 有退回、销毁、补发规则 |

### 1.6 P0-A 的 7 天执行方案

| 天数 | 动作 | 完成标准 |
|---:|---|---|
| Day 1 | 选 5 个轻小无电池 SKU，计算 CNY 成本、USD 售价、物流费、手续费 | 每个 SKU 有一张成本卡 |
| Day 1 | 注册/整理 PayPal Business，准备 PSP 申请资料 | 至少有 1 条能生成付款请求的路径 |
| Day 2 | 建一个极简商品页或 Notion/独立页，补齐退款、物流、隐私、联系方式 | 客户能看懂商品、价格、发货地、时效和退款规则 |
| Day 2 | 找 3 个中国到 Kazakhstan 可追踪物流报价 | 每个报价都有时效、价格、追踪、禁运和赔付说明 |
| Day 3 | 发 1-2 个内部测试包裹到 Kazakhstan 地址，或找物流商提供近期同线路妥投单号样例 | 能验证线路不是口头承诺 |
| Day 3-5 | 在 Instagram/TikTok/Telegram/WhatsApp 做小规模内容和私域触达 | 不承诺本地现货，不承诺货到付款 |
| Day 4-7 | 拿到第一笔陌生客户在线付款后，从中国发货并留证据 | 订单台账记录付款、发货、物流、客服节点 |
| 妥投后 | 做客户确认和复盘 | 有客户确认收货或物流妥投截图 |

### 1.7 P0-A 的 Go/No-Go

| 结果 | 判断 | 下一步 |
|---|---|---|
| 有陌生客户付款，包裹妥投 | P0-A 成功 | 复制 5-10 单，再决定是否补本地仓配 |
| 有付款，物流卡住 | 半成功 | 先修物流，不急着扩 SKU |
| 有询盘，无付款 | 支付/信任/价格未过关 | 换收款方式、增强页面信任、调价 |
| 有付款，客户因税费或时效退款 | 履约承诺有问题 | 改话术、改物流、筛城市 |
| 7 天无有效付款 | 获客或选品未过关 | 先调内容和 SKU，不要直接搭本地仓 |

### 1.8 本地路线后置说明

如果后面选择本地收款和本地派送路线，你们现在明确没有本地派送方式，也没有本地收付款方式。所以本地路线当前不是 P0，而是：

```text
P0-B 前置：本地收款和本地履约能力补齐
```

这条路线不是这次的初期主线，但后面做 Kaspi、本地复购、快速配送和退换货时仍然要补。

## 2. 关键修正：Kaspi 不是 P0 前置条件

Kaspi 很重要，但不能把它放在第一单之前。

我复核了 Kaspi 官方合作伙伴指南。成为 Kaspi.kz 商店合作伙伴，需要先接入 Kaspi Pay，并且在最近 2 个月内至少 20 天通过 Kaspi Pay 收款。官方说明还写明，企业需要销售实物商品，服务类应去“Объявления”服务。也就是说，如果你们现在没有本地主体、Kaspi Pay 和收款记录，Kaspi 不能作为 P0 的唯一入口。

正确排序：

| 阶段 | Kaspi 的位置 | 动作 |
|---|---|---|
| P0 | 研究和准备，不作为收款/履约入口 | 跑跨境直发成交，同时确认本地主体、Kaspi Pay、收款记录要求 |
| P1 | 入驻任务 | 用已成交 SKU 去申请或通过合作方上架 |
| P2 | 主平台运营 | 做评价、排名、库存周转、售后、复购 |

## 3. P0 的硬边界

### 3.1 P0 算成功的标准

同时满足以下条件才算 P0 成功：

| 条件 | 标准 |
|---|---|
| 客户来源 | 陌生客户，来自 OLX、Instagram、Telegram、WhatsApp、线下转介绍等真实获客渠道 |
| 付款 | PayPal、国际卡、PSP payment link 等在线付款之一 |
| 交付 | 中国端发货后有 tracking number，最终物流妥投或客户明确确认收货 |
| 证据 | 有商品页截图、聊天截图、付款凭证、发货凭证、物流轨迹、收货/交付凭证 |
| 复盘 | 订单记录能回溯 SKU、渠道、价格、币种、汇率、物流、售后 |

### 3.2 不算 P0 成功的情况

| 情况 | 为什么不算 |
|---|---|
| 朋友帮买 | 不是陌生客户需求 |
| 合作方自己买 | 不能证明市场需求 |
| 只加购物车 | 没有付款 |
| 只问价 | 没有交易 |
| 全额 COD | 客户没有提前承担付款风险，拒收风险没有验证 |
| 免费送样 | 没有验证支付意愿 |
| 只拿到批发口头意向 | 不是 P0 的 C 端闭环 |

## 4. 团队角色和责任

一个人可以兼多个角色，但每个责任必须有人负责。

| 角色 | 负责什么 | 每日交付物 |
|---|---|---|
| 项目负责人 | 阶段目标、SKU 取舍、预算、复盘、推进卡点 | 每日复盘结论和次日任务 |
| 供应链负责人 | 拿货、质检、包装、中国端发货 | SKU 成本表、质检记录、发货记录 |
| 物流负责人 | 中国到 Kazakhstan 直发报价、测试包、追踪、异常处理 | 物流报价表、tracking、妥投/异常记录 |
| 支付负责人 | PayPal/PSP 申请、测试付款、手续费、退款、拒付处理 | 付款链接、到账截图、手续费和退款记录 |
| 客服负责人 | 俄语/当地语言承接、报价、催付、售后 | 聊天记录、客户问题表、付款转化表 |
| 内容负责人 | 图片、短视频、商品标题、商品描述、上架 | 每个 SKU 的图文素材包 |
| 数据负责人 | 订单台账、渠道数据、复盘指标 | 每日数据表和异常记录 |
| 合规负责人 | HS code、认证、禁售、平台规则、税费假设 | SKU 合规预筛表 |
| 本地负责人 | 仅 P0-B/P1 需要：本地收款、轻仓、派送、自提、退换货地址 | 本地合作方筛选表和测试记录 |

最低配置：

```text
中国端 3 人：供应链/物流 + 支付/数据 + 内容/客服
本地端 0 人：P0-A 不配置
本地负责人：P0-B/P1 再补
```

## 5. 执行前 48 小时必须确认的事

这部分没确认，不要投广告，不要大量备货。

### 5.0 P0-A：跨境直发 48 小时检查表

| 项目 | 最低完成标准 |
|---|---|
| 收款 | 至少 1 条真实可用路径，能生成付款链接或 checkout，并完成内部小额测试 |
| 退款 | 知道如何退款、多久到账、手续费是否退回 |
| 物流 | 至少 3 个中国到 Kazakhstan 可追踪报价，且发出 1-2 个测试包或拿到同线路妥投样例 |
| 商品 | 5 个 SKU 都完成禁运/敏感品预筛，不含电池、液体、无线发射等高风险点 |
| 页面 | 商品页写清 Ships from China、预计时效、税费承担、退款规则、联系方式 |
| 客服 | 俄语/英语付款话术、物流话术、退款话术都能直接复制发送 |
| 台账 | 每单能记录 SKU、渠道、客户城市、付款方式、物流商、tracking、售后状态 |
| 合规 | 申报品名、数量、价值如实填写，不低报、不错报 |

### 5.1 后置方案：本地能力补齐 SOP

这部分暂时不作为初期主线。只有当你们决定从“纯跨境直发”升级到本地仓配、本地平台、本地复购时，再执行这一段。

本地能力补齐的完成标准：

```text
1 条可测试收款路径
+ 1 条可测试本地交付路径
+ 1 个本地库存/退货接收点
+ 1 套对账和责任边界
```

这个阶段不追求新增订单，只追求把本地电商基础设施补齐。

#### 5.1.1 收款路径怎么补

优先顺序：

| 优先级 | 路径 | 需要什么 | 验证标准 | 风险 |
|---:|---|---|---|---|
| 1 | 本地可信合作方代收 | 合作方可收 Kaspi/银行转账/本地电子钱包 | 内部小额付款到账，并能回款给你们 | 资金和对账依赖对方 |
| 2 | 本地个体/公司主体接 Kaspi Pay | 本地 IP/TOO、银行账户、Kaspi Pay | 完成注册并能收测试款 | 周期可能超过 P0 时间 |
| 3 | Halyk EPAY 或其他 payment link | 本地商户资料和银行账户 | 生成链接，完成测试付款和退款 | 接入门槛和审核资料更重 |
| 4 | 跨境收款后本地交付 | 客户愿意用国际卡/其他方式付款 | 陌生客户能顺利支付 | 本地信任和支付体验弱 |
| 5 | 全额 COD | 不需要预付款 | 只能作为交付测试 | 不算 P0 成功 |

如果后面要做本地路线，最务实的路线是先找本地可信合作方代收。不要一开始把目标定成“马上自己开 Kaspi 店”，那会被主体、收款记录和平台审核卡住。

## 6. SKU 选择 SOP

### 6.1 P0 SKU 原则

P0 只选能快速验证交易链路的商品，不选“看起来市场很大但售后复杂”的商品。

必须满足：

| 标准 | 要求 |
|---|---|
| 轻小 | 单件最好小于 500g |
| 低损耗 | 不易碎、不漏液、不变质 |
| 低合规 | 无电池、无蓝牙、无 Wi-Fi、无射频、无插电 |
| 易展示 | 一张场景图或 15 秒视频能讲清价值 |
| 易替换 | 坏了可以补发，不需要维修 |
| 通用性 | 不强依赖车型、系统、语言、尺码 |
| 低客诉 | 不涉及安全、医疗、功效承诺 |

### 6.2 默认首批 5 个 SKU

| SKU | 首批数量 | 核心卖点 | 主要风险 | P0 处理 |
|---|---:|---|---|---|
| 出风口车载手机支架 | 10-20 | 导航稳定、安装简单 | 兼容部分出风口差 | 商品页写清适配类型 |
| 仪表台/吸盘手机支架 | 10-20 | 不占出风口、视角灵活 | 吸盘稳定性 | 实拍颠簸测试视频 |
| 座椅缝隙收纳盒 | 10-20 | 车内整洁、防掉落 | 车型缝隙差异 | 标注尺寸，提醒测量 |
| 理线夹/挂钩组合 | 20-50 | 便宜、轻小、可加购 | 客单低 | 做套装，不单独卖 |
| 防滑垫/临时停车牌 | 20-50 | 低价、易理解 | 同质化 | 用真实车内场景图提高信任 |

### 6.3 明确暂缓的 SKU

| SKU | 暂缓原因 | 什么时候再看 |
|---|---|---|
| 充电宝 | 电池运输、UN38.3、MSDS、售后和安全责任 | P1 后，有合规文件和退货能力 |
| TWS 耳机 | 蓝牙、电池、质量波动、退换货高 | P1/P2，有稳定供应商和 EAC 路径 |
| 车载充电器 | 发热、电气安全、认证、客诉风险 | P1 后，拿到合规资料 |
| 无线 CarPlay | 兼容性复杂、固件问题、退货率风险 | P2 后，有技术客服 |
| 智能手环/手表 | App、语言、蓝牙、电池、售后 | P2 后 |
| 小家电 | 插头、电压、重量、认证、破损 | P2 后 |
| 香薰液体 | 液体运输、泄漏、气味主观 | 暂不做 |
| 功能性汽车配件 | 安全责任、车型匹配复杂 | 暂不做 |

### 6.4 SKU 打分表

每个候选 SKU 先打分，低于 70 分不进入 P0。

| 指标 | 分值 | 评分说明 |
|---|---:|---|
| 需求直观 | 15 | 用户一眼能不能懂 |
| 展示容易 | 10 | 能不能用短视频讲清 |
| 轻小易发 | 15 | 重量、体积、破损风险 |
| 合规简单 | 20 | 是否避开电池、无线、插电、液体 |
| 售后简单 | 15 | 坏件能否低成本补发 |
| 供应稳定 | 10 | 是否能 7 天内补货 |
| 毛利空间 | 10 | 扣除物流后是否有空间 |
| 差异化 | 5 | 是否能靠套装、图文、场景做差异 |

## 7. 供应链和质检 SOP

### 7.1 拿货前问供应商

| 问题 | 为什么 |
|---|---|
| 是否有现货 | P0 不等长交期 |
| 最小起订量多少 | 控制首批库存 |
| 是否支持混批 | 方便测多个 SKU |
| 是否能提供中性包装 | 避免包装语言和品牌不适配 |
| 是否能补货 | P1 要复制 |
| 是否有质检报告/认证资料 | P1/P2 可能用到 |
| 是否支持坏件补发 | 降低售后成本 |
| 是否有同款多档质量 | 避免拿低质版本 |

### 7.2 到货质检

每个 SKU 抽检至少 20%，不足 10 件的全部检查。

| 检查项 | 通过标准 |
|---|---|
| 外观 | 无明显划痕、污渍、变形 |
| 功能 | 支架能夹住手机，收纳盒能正常安装 |
| 尺寸 | 和详情页一致 |
| 包装 | 不破损，不露中文批发标签 |
| 配件 | 螺丝、贴片、说明卡齐全 |
| 场景测试 | 能在真实车内安装并拍摄 |

质检后给每个 SKU 留：

```text
实物照片
包装照片
尺寸照片
安装视频
问题件照片
```

## 8. 报价和利润测算 SOP

### 8.1 单 SKU 报价表

| 场景 | 规则 |


## 11. 收款 SOP

### 11.1 收款优先级

| 优先级 | 方式 | 适用条件 | 风险 |
|---:|---|---|---|
| 1 | PayPal Business 发票/付款链接/Checkout | 初期没有本地人、没有本地主体 | Kazakhstan 用户接受度要实测，注意风控和提现 |
| 2 | 国际卡收单 PSP | 能提交主体、商品页、物流/退款政策 | 审核资料更重，可能被风控补件 |
| 3 | Shopify/SHOPLINE/Shoplazza 等建站工具 + 支付插件 | 想让客户看到更正式的 checkout | 月费、插件费和配置时间更高 |
| 4 | Kaspi Pay / Halyk EPAY / 本地银行转账 | 后续有本地合作方或本地主体 | P0-A 不依赖，放到 P0-B/P1 |
| 5 | 全额 COD | 只作为观察 | 不计入 P0-A 成功 |

Halyk EPAY 官方说明其提供面向商户的在线支付能力，并支持银行卡等付款方式。它更适合本地路线 P0-B/P1 接入，不作为“不找本地人”阶段的第一收款方式。

### 11.2 收款测试

上线前必须完成一笔内部测试：

```text
测试金额：小额
付款人：团队成员或本地测试人
收款方式：真实 P0-A 计划使用方式
记录：付款截图、到账截图、手续费、到账时间、退款方式
```

如果收款测试失败：

```text
暂停上架和投放
先修收款
不要用“客户可以先下单，付款后面再说”的方式硬跑
```

| 售后 | 售后率可控，能定位问题 |

## 20. 风险清单

| 风险 | 触发点 | 处理 |
|---|---|---|
| 收款不可控 | 款在合作方账户，无法对账 | 每日对账，合同写清 T+几天结算 |
| 库存不可控 | 货在本地节点但无台账 | 每日库存照片和出入库记录 |
| 价格失真 | 没算税费、物流、售后 | 单 SKU 成本表强制填写 |
| 合规误判 | 电子品类没有 EAC/电池资料 | P0 避开复杂 SKU |
| 客服失控 | 多人乱报价 | 统一报价表和话术 |
| 渠道误判 | 只看浏览，不看付款 | 只用付款和有效询盘判断 |
| 合作方风险 | 本地资源全在一个人手里 | 至少准备备选合作方 |
| 汇率风险 | KZT 波动影响毛利 | 报价预留 5%-10% 汇率缓冲 |

## 21. 今天就能开始的任务清单

如果你们今天要开干，按这个顺序：

1. 暂停“先找本地人、本地仓、本地派送”的动作，把它改成后置路线。
2. 确认一个能最快真实收款的路径：PayPal Business、PayPal 发票/付款链接，或已能开通的 PSP。
3. 同时申请 1-2 个国际卡收单 PSP，准备主体、商品页、物流政策、退款政策、联系方式。
4. 选 5 个轻小无电池 SKU，优先车内支架、收纳、内饰、小件手机周边。
5. 找 3 个中国到 Kazakhstan 可追踪物流报价：邮政/ePacket、菜鸟/跨境小包、中哈专线、商业快递。
6. 做 1-2 个物流测试包，或者要求物流商提供近期 Kazakhstan 同线路妥投单号样例。
7. 建一个极简商品页，写清 Ships from China、15-35 天预计时效、税费由买家承担、可追踪发货。
8. 把报价币种设成 USD，必要时附 KZT 参考价；内部用 CNY 成本表核算。
9. 发 Instagram/TikTok/Telegram/WhatsApp 内容，只承诺中国直发，不承诺本地现货和货到付款。
10. 拿到陌生客户在线付款后，从中国发货并记录付款、物流、客服、妥投证据。
