# 哈萨克斯坦外部报告导读

更新时间：2026-06-20
适用范围：中亚五国出海电商 MVP，重点校准哈萨克斯坦首站策略。
资料来源：本目录下 `外部资料/2026-06-哈萨克斯坦/` 的 3 份 PDF。

## 先说结论

这 3 份报告会改变我们之前 SOP 的权重排序：

| 决策项 | 原先默认 | 读完报告后的修正 |
|---|---|---|
| 首站国家 | Kazakhstan | 继续确认 Kazakhstan 首站合理，原因是 GDP、支付、电商和物流基础最好 |
| 首站城市 | Almaty | 继续确认 Almaty 优先，商务、金融、交通、华人和中资机构更集中 |
| 核心平台 | OLX.kz + Instagram + Telegram/WhatsApp | 如果能解决本地卖家主体和收款，Kaspi 应升为主战场；不能马上入驻时，OLX/Instagram/Telegram 继续用于 P0 测闭环 |
| 首批 SKU | 车载支架、收纳、理线、通用手机支架 | P0 仍优先无电池、无无线、低认证风险 SKU；P1 再评估移动配件、智能穿戴、小家电 |
| 消费电子方向 | 保守避开电池/蓝牙/Wi-Fi/RF | 机会存在，但不能一股脑上。TWS、充电宝、智能手环、小家电需要 EAC/税务/售后准备后再进 |
| 中亚扩张 | 五国长期都看 | 推荐路径更清晰：哈萨克斯坦先行，再乌兹别克斯坦，再吉尔吉斯斯坦；不要一开始五国铺开 |

执行上，P0 的目标仍然不是利润，而是拿到首批订单并跑通交易链路。报告给出的最大价值是：证明哈萨克斯坦不是随便选的，同时把 Kaspi、EAC、VAT、汇率、合作伙伴这些关键坑提前抬出来。

## 报告一：宏观画像报告怎么读

这份报告回答的是：为什么中亚先做哈萨克斯坦，以及哈萨克斯坦能不能作为区域入口。

关键点：

| 维度 | 报告信息 | 对 MVP 的含义 |
|---|---|---|
| 区域体量 | 中亚五国总人口约 8220 万，2024 年 GDP 合计约 4830 亿美元；哈萨克斯坦占区域 GDP 约 57.8% | 先做哈萨克斯坦不是因为人口最多，而是因为支付、电商、购买力和基础设施更成熟 |
| 物流地位 | 中欧班列超过 80% 经由哈萨克斯坦过境，中国-哈萨克斯坦铁路货运增长明显 | 从中国供货到哈萨克斯坦有通道基础，但 P0 不要直接假设时效稳定，仍要小批量测 |
| 支付 | 哈萨克斯坦非现金支付比例高，Kaspi Pay 处理大量非现金交易，城市 COD 率较低 | 城市订单可以优先围绕在线支付/本地转账设计，不必默认只靠货到付款 |
| 电商 | 哈萨克斯坦 2024 电商市场规模约 62 亿美元，Kaspi 份额最高，Wildberries/Ozon 次之 | 平台优先级要从“泛渠道试卖”逐步转向“Kaspi 主场验证” |
| 语言 | 俄语仍强，哈萨克语在身份和本地化上越来越重要 | 商品标题、详情页、包装和客服话术至少准备俄语；品牌长期要补哈萨克语版本 |
| 营销 | Instagram/Telegram/TikTok 是核心信息源，KOL/口碑高于传统广告 | 不要一开始砸传统广告，先做真实买家反馈、短视频测评、KOL 小样 |
| 区域扩张 | 哈萨克斯坦到吉尔吉斯斯坦同属 EAEU，扩张阻力低；到乌兹别克斯坦有非 EAEU 壁垒 | P0/P1 不碰五国全铺；先哈萨克斯坦，后续再分 EAEU 与非 EAEU 两条线 |

我的判断：

哈萨克斯坦适合作为第一站，但不是因为它“简单”。它的优势是基础设施和支付成熟，问题是平台集中、合规更严、竞争更正规。对小白团队来说，正确打法不是一上来做“大跨境商城”，而是先在 Almaty 用小 SKU 跑交易链路，再逐步接入 Kaspi 和本地履约。

## 报告二：商务部国别指南怎么读

这份报告不是用来选品的，是用来查底线的：税、海关、企业设立、合作伙伴风险、投资风险。

和我们最相关的内容：

| 维度 | 报告信息 | 对 MVP 的含义 |
|---|---|---|
| 城市 | 阿拉木图是哈萨克斯坦最大城市，也是经济、金融、交通中心 | 首批测试城市继续锁 Almaty，不要一开始做全国履约 |
| 互联网 | 截至 2024 年初，互联网用户约 1890 万，占全国人口 92.3%；社交网络活跃用户约 1400 万 | 电商获客基础够，社媒种草可以作为低成本测试方式 |
| 电商政策 | 哈萨克斯坦有《2025 年前电子商务发展路线图》，方向包括扩大电商出口、吸引企业参与基础设施建设、加强消费者权益保护 | 政府方向支持电商，但也意味着消费者保护和税务合规会越来越重要 |
| 进口税制 | 哈萨克斯坦在 EAEU 框架下执行统一进口关税税率 | 做 SKU 前要查 HS code，不要凭“别人能卖”判断自己能进口 |
| VAT | 进口增值税为 12%；外国互联网公司通过电商平台或网站销售商品或服务时也涉及 12% VAT | 报价表必须单独列税费假设，否则利润测算会失真 |
| 海关 | 货物在到货终点口岸办理清关手续 | P0 需要找清关/物流服务商确认小批量模式，不要先压大货 |
| 合作伙伴 | 当地伙伴能解决关系、法律、执法、资源问题，但也有资信和过度依赖风险 | 本地合作方必须做尽调，合同里写清收款、库存、退货、责任边界 |
| 汇率 | 坚戈受石油、卢布、外贸影响，存在不可预测汇率风险 | 首批可以不追利润，但记录每单汇率和币种；P2 之前必须有汇率缓冲 |

我的判断：

这份报告对我们的提醒很直接：本地合作方很重要，但不能把命门全交给对方。P0 如果借本地人收款或代发货，必须同步保留订单台账、库存台账、回款记录、聊天证据和退货记录。否则首批订单跑通了，也无法复制。

## 报告三：消费电子品类机会报告怎么读

这份最贴近你们现在想做的“消费电子 + 车内用品”。

报告结论：

| 品类 | 报告倾向 | 我的执行判断 |
|---|---|---|
| 移动配件/周边 | 第一优先级 | 可以做，但要拆。无电池、无无线、无认证压力的先做；TWS、充电宝、车充放到 P1/P2 |
| 智能穿戴 | 第一优先级 | 市场有机会，但涉及无线、电池、EAC、售后，P0 不建议作为第一批闭环 SKU |
| 小家电 | 第一优先级 | 市场有机会，但重量、认证、售后、插头标准、退货成本都更高，P0 不建议 |
| 智能家居 | 第二优先级 | 可作为 P1/P2 方向，尤其安防/照明；P0 暂缓 |
| 家电 | 第二优先级 | 更适合本地渠道或分销，不适合首批小白 MVP |
| 笔记本整机 | 第三优先级 | 不做整机，最多测电竞外设 |
| 无人机 | 第三优先级 | 暂缓，监管、品牌垄断和售后风险都高 |

报告推荐“Kaspi 为主，多平台协同”。这个方向对，但要拆成两个阶段：

| 阶段 | 平台动作 | 为什么 |
|---|---|---|
| P0 | OLX.kz、Instagram、Telegram/WhatsApp、线下熟人转介绍先跑订单；同步研究 Kaspi 入驻要求 | P0 的目标是闭环，不是平台规模；Kaspi 如果需要本地主体/资料/收款，可能卡住 |
| P1 | 有本地主体或可靠合作方后，优先上 Kaspi Marketplace；Wildberries/Ozon 做补充 | Kaspi 是主战场，评价体系和支付心智强，适合复制闭环 |
| P2 | 做稳定 SKU、评价、售后仓和复购；再考虑智能穿戴/小家电 | 这时才有能力处理认证、退换货和库存周转 |

## 对我们现有 SOP 的具体改法

### 1. P0 首批 SKU 不要被“消费电子机会”带偏

P0 目标是首批订单闭环，所以优先：

| 可进入 P0 | 原因 |
|---|---|
| 车载手机支架 | 贴近你们原方向，低客单、低售后、无电池 |
| 车内收纳盒/挂钩/理线夹 | 非电子，运输和合规压力小 |
| 手机壳/膜/普通支架 | 移动配件需求大，试错成本低 |
| 车载防滑垫/临时停车牌/内饰小件 | 低价、轻小、适合测图文和短视频 |
| 电竞外设小件，如鼠标垫、手柄收纳、理线 | 可蹭消费电子需求，但不碰复杂认证 |

P0 暂缓：

| 暂缓品类 | 原因 |
|---|---|
| 充电宝 | 电池运输、认证、售后、召回风险 |
| TWS 耳机 | 蓝牙/无线/电池/EAC/退换货压力 |
| 车载充电器 | 电子安全、认证和质量责任 |
| 智能手环/智能表 | 电池、无线、App、售后、真假参数风险 |
| 小家电 | 重量、插头、电压、认证、退货成本 |
| 无人机及配件 | 监管和售后复杂，P0 不碰 |

### 2. 渠道 SOP 要改成“双轨”

| 轨道 | 目标 | 平台 |
|---|---|---|
| 闭环轨 | 快速拿首批订单 | OLX.kz、Instagram、Telegram/WhatsApp、线下熟人介绍 |
| 规模轨 | 为 P1/P2 做准备 | Kaspi Marketplace、Wildberries、Ozon、Sulpak/Technodom |

闭环轨先回答：有没有人问、有没有人付钱、怎么交付、怎么售后。
规模轨再回答：能否入驻、手续费多少、回款周期多久、评价怎么积累、退货规则是什么。

### 3. 订单台账必须从第一单开始

每一单都记录：

| 字段 | 为什么 |
|---|---|
| 获客渠道 | 判断哪个渠道真能出单 |
| SKU | 后续选品复盘 |
| 采购价、售价、币种、汇率 | P0 不求利润，但 P2 必须回看 |
| 支付方式 | 判断本地收款可复制性 |
| 履约方式 | 判断能不能稳定发货 |
| 客户问题 | 反推详情页和客服话术 |
| 退换货/售后 | 判断 SKU 是否能复制 |

## 新名词解释

| 名词 | 是什么 | 在本项目里怎么用 |
|---|---|---|
| Kaspi | 哈萨克斯坦最核心的金融、电商和生活服务平台之一 | P1/P2 的主战场候选，P0 先研究入驻和收款门槛 |
| Kaspi Pay | Kaspi 的支付能力 | 如果有本地主体/合作方，可作为本地收款方案之一 |
| EAEU | 欧亚经济联盟，成员包括哈萨克斯坦、俄罗斯、白俄罗斯、吉尔吉斯斯坦、亚美尼亚 | 影响关税、清关、认证和区域扩张 |
| EAC | EAEU 的产品合格标志 | 消费电子合法销售绕不开，P0 应先避开复杂认证 SKU |
| TR CU | 关税同盟技术法规 | 判断电子产品是否需要 EMC、RoHS 等合规要求 |
| VAT | 增值税 | 哈萨克斯坦相关口径常见 12%，报价和利润测算必须单列 |
| COD | 货到付款 | 哈萨克斯坦城市 COD 依赖低于很多中亚农村市场，但不能完全忽略 |
| KD 组装 | 零部件进口后在当地组装 | 适合后期家电/高价值电子产品降税，不是 P0 动作 |
| HS code | 商品海关编码 | 每个 SKU 做进口、税率、限制判断前必须查 |

## 给小白团队的下一步

1. 先不改 P0 的“轻小、无电池、低认证”原则。
2. 把 SKU 池分成三层：P0 立刻测、P1 合规后测、P2 有本地售后后测。
3. 7 天内完成 Kaspi 入驻条件确认：主体、收款、保证金、佣金、回款周期、退货规则。
4. 7 天内找 2 个本地合作方候选：一个偏收款/平台，一个偏仓配/退货。
5. 每个候选 SKU 先做 10 条本地化标题和 5 张主图测试，不要先压货。
6. 第一批只备 20-50 件以内，按订单反馈补货。

## 本次导读的限制

- 我只基于这 3 份 PDF 做阅读和整理，没有在本次任务里重新联网复核所有数字。
- 报告中 Kaspi 市占率有 60% 和 70% 以上两种表述，执行时按“绝对主平台”理解，但具体数值要再查。
- 消费电子机会不等于 P0 可做。带电池、无线、插电、App、强售后的 SKU 都要后置。
- 商务部国别指南是 2024 版，法律、税率、平台规则和认证周期执行前必须再确认。
