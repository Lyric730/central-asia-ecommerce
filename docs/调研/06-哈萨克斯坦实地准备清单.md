# 哈萨克斯坦实地准备清单：以开 Kaspi 网店为目标

复核日期：2026-06-29  
适用对象：即将前往 Kazakhstan / Almaty 的本地合伙人  
目标：最低跑通本地收款 + 本地履约 + Kaspi Pay 真实经营记录；最高进入 Kaspi Marketplace 完整体经营。

## 0. 先说结论

这次实地考察不能只做“市场看看”和“找几个渠道”。如果目标是开 Kaspi 网店，实地合伙人要围绕 4 个硬前置办事：

```text
本地可控主体 / 可控合作方
+ Kaspi Pay
+ 近 2 个月内至少 20 天真实 Kaspi Pay 收款
+ 本地仓配、退货、商品合规资料
-> 才进入 Kaspi 店铺申请
```

Kaspi 官方当前规则里，成为 Kaspi.kz 商店伙伴需要先是 Kaspi Pay 伙伴，并且最近 2 个月内至少 20 天通过 Kaspi Pay 收款。销售给自己、公司员工以及客户退货不计入。连接 Kaspi 店铺只能由公司负责人操作。被拒后 1 个月才能重新提交。[R1][R2][R3][R4]

另外，2026-06-15 后新接入 Kaspi 店铺的卖家，自提功能不可用；新卖家要准备自己配送、Kaspi Delivery 或 Express delivery，而不是只准备一个自提点。[R5][R6]

## 1. 路径定义

| 路径 | 目标 | 什么时候用 | 核心风险 |
|---|---|---|---|
| A. 本地合作方路径 | 用 Kazakhstan 本地 IP/TOO/店铺资源先开 Kaspi Pay，跑 20 天真实收款，再申请 Kaspi 店 | MVP 最快 | 钱、库存、账号都可能被合作方卡住 |
| B. 自有 Kazakhstan TOO/LLP 路径 | 注册或控制自己的 Kazakhstan 法人，自己开 Kaspi Pay 和 Kaspi 店 | 完整体主路径 | 周期更长，税务、会计、银行、法务成本更高 |
| C. 非居民公司直连路径 | 用中国/境外公司尝试接 Kaspi Pay / Kaspi 店 | 并行探索，不作主线 | Kaspi 对非居民 Kaspi Pay 逐案决定，资料包和审核不确定 [R7][R8] |

执行判断：

```text
MVP 最低要求：A 路径可跑，但必须把账号、收款、库存、回款、退货边界写清。
完整体要求：B 路径优先，A 路径只能作为过渡或渠道合作。
C 路径可以问，但不能把项目成败押在 C 路径。
```

## 8. Go / No-Go 决策

### 8.1 可以继续推进 Kaspi

满足以下条件，才进入 20 天 Kaspi Pay 真实收款周期：

```text
有可控主体或高可信合作方
+ Kaspi Pay 已开通或资料缺口明确
+ 有 2 个以上本地 +7 号码
+ 有轻仓/发货/退货地址
+ 有至少 5 个低风险 SKU 的资料包
+ 有订单/收款/库存台账
+ 有会计或税务服务商确认收款和报税处理
```

### 8.2 暂停 Kaspi，继续跑非 Kaspi 本地闭环

出现任一情况，Kaspi 暂停，不要硬冲：

```text
找不到可控主体，也无法注册自有主体。
Kaspi Pay 开不了，且拒绝原因不清楚。
本地合作方只愿意代收钱，不愿意给流水/对账/合同。
无法准备仓配和退货地址。
20 天收款计划靠刷单、自买、员工付款。
首批 SKU 涉及带电/无线/电池但没有认证路径。
```

### 8.3 直接 No-Go

```text
本地合作方要求账号、库存、资金都归他控制。
对方不愿意提供主体证明、税务状态、Kaspi Pay 权限截图。
对方要求用灰色清关、低报、错报品名。
对方承诺“包开 Kaspi 店”，但不能解释 20 天 Kaspi Pay 收款规则。
```

## 9. 合作方尽调表

| 字段 | 要填什么 | 通过标准 |
|---|---|---|
| 主体类型 | IP / TOO / 个人 / 物流商 / 店铺 | IP/TOO 优先 |
| 主体编号 | BIN / IIN | 能核验 |
| 负责人 | 姓名、电话、是否本人可操作 Kaspi Pay | 必须本人配合 |
| Kaspi Pay 状态 | 未开通 / 已开通 / 有收款历史 / 接近 20 天 | 已开通优先 |
| 税务状态 | 是否有欠税/账户限制 | 无欠税/无限制 |
| 可否收款 | Kaspi QR、远程付款、Smart POS、银行转账 | 至少 Kaspi Pay |
| 可否对账 | 每日流水、订单号、回款周期 | 必须 |
| 可否仓储 | 件数、面积、开放时间 | 至少 50-100 件 |
| 可否配送 | 自送、跑腿、Kaspi Delivery、Express | 至少 1 条稳定路径 |
| 可否退货 | 地址、拍照、检测、补发 | 必须 |
| 费用 | 收款费、仓储费、出库费、配送费、服务费 | 写成表 |
| 退出机制 | 终止后资金/库存/账号/客户如何交接 | 必须 |
