# 06 复盘和阶段门槛

## P0 通过门槛

必须同时满足：

| 项 | 标准 |
|---|---|
| 客户 | 陌生真实客户 |
| 付款 | 全额预付款、平台内付款、本地转账或 payment link |
| 交付 | 商品送达或自提完成 |
| 证据 | 渠道、聊天、付款、发货、签收、售后记录完整 |
| 商品 | 无硬淘汰风险 |
| 售后 | 客诉有处理记录 |

P0 不要求利润为正，但必须记录真实成本。

## P0 失败类型

| 失败类型 | 含义 | 下一步 |
|---|---|---|
| 有曝光无咨询 | 素材/价格/渠道错 | 换素材、换卖点、换渠道 |
| 有咨询无付款 | 信任/支付/价格/配送承诺弱 | 优化付款方式、发货承诺、客户证明 |
| 有付款交付失败 | 履约链路不成立 | 修履约，不急着扩 SKU |
| 售后爆炸 | SKU 不适合 P0 | 淘汰 SKU |
| 只靠熟人成交 | 市场验证无效 | 重新找陌生渠道 |

## P1 进入门槛

满足任一：

- 同一国家同一链路完成 10 个陌生客户付款订单。
- 2 个 SKU 各完成 3 个陌生客户付款订单。
- 同一 SKU 在 2 个渠道均完成陌生客户付款订单。

同时必须满足：

- 每单证据完整。
- 平均客服耗时可接受。
- 补发/退款成本可量化。
- 收款和对账没有重大漏洞。
- 履约时效有稳定预期。

## P1 工作重点

| 模块 | 动作 |
|---|---|
| 渠道 | 固化最佳渠道和话术 |
| 商品 | 保留成交 SKU，淘汰高咨询低付款 SKU |
| 内容 | 把有效素材标准化 |
| 支付 | 从个人/合作方临时收款转向商户账户/主体 |
| 履约 | 从人工发货转向轻仓/固定 3PL |
| 平台 | 准备 Kaspi/Uzum/Olcha/Ozon/WB/Flip 等入驻 |

## P2 进入门槛

| 项 | 标准 |
|---|---|
| 时间 | 连续 30 天有订单 |
| 利润 | 扣除广告、补发、退款、客服、仓配后毛利为正 |
| 售后 | 退款/补发率低于内部阈值 |
| 供应链 | 可稳定补货 |
| 履约 | 时效和丢件率稳定 |
| 平台 | 至少 1 个可复制渠道有稳定订单 |

## P3 进入门槛

| 项 | 标准 |
|---|---|
| SKU | 已有 C 端验证或明确 B2B 需求 |
| 报价 | 有批发价、最小起订量、交期 |
| 样品 | 可提供样品和规格资料 |
| 收款 | 明确预付款/账期/风险控制 |
| 售后 | 明确坏件、补件、退货规则 |

## 周复盘

每周固定复盘：

- 哪个国家数据最好。
- 哪个 SKU 询盘最高。
- 哪个 SKU 付款最高。
- 客户最常问什么。
- 哪个渠道客服耗时最低。
- 哪个承诺造成售后风险。
- 下周保留/淘汰/新增什么。
